## Понятие потока и файла

| Термин | Определение |
|--------|-------------|
| **Поток (stream)** | Абстрактный логический интерфейс, через который программа либо **порождает** (вывод), либо **потребляет** (ввод) данные. Все потоки ведут себя одинаково, независимо от физического устройства. |
| **Файл (file)** | Конкретное физическое устройство или место хранения: дисковый файл, экран, клавиатура, принтер, сетевой порт и т.п. |

> 💡 **Главная идея:** Однотипные операции (например, `<<`) работают с любым устройством. Код, выводящий на экран, без изменений выведет в файл, если сменить поток.

---

## Виды потоков

| Тип | Особенности |
|-----|-------------|
| **Текстовый** | Оперирует символами. Могут происходить преобразования (например, `\n` превращается в `CR+LF` в Windows). Нет строгого соответствия между отправленными и фактически записанными данными. |
| **Двоичный (бинарный)** | Данные передаются «как есть», без преобразований. Используется для любых типов данных. |

---

## Текущая позиция в потоке

**Текущая позиция** – это место в файле/устройстве, с которого будет выполнена следующая операция чтения или записи. Например, если файл имеет размер 100 байт и вы уже прочитали 50 байт, то текущая позиция – байт №50.

---

## Встроенные стандартные потоки

При старте программы автоматически открываются четыре потока:

| Имя | Назначение | Буферизация |
|-----|------------|-------------|
| `cin`  | Стандартный ввод (обычно клавиатура) | – |
| `cout` | Стандартный вывод (обычно экран) | – |
| `cerr` | Вывод ошибок | **Нет** (вывод немедленный) |
| `clog` | Вывод ошибок / логов | **Да** (вывод при заполнении буфера) |

Для работы с широкими символами (Unicode) существуют версии: `wcin`, `wcout`, `wcerr`, `wclog`.

## Иерархия классов ввода/вывода

Вся система базируется на шаблонных классах (для обычных и широких символов). Для повседневного использования применяются **символьные** (8-битные) классы:

| Шаблонный класс | Символьный класс | Назначение |
|-----------------|------------------|------------|
| `basic_streambuf` | `streambuf` | Низкоуровневый буфер, работа с памятью и устройствами (редко используется напрямую) |
| `basic_ios`      | `ios`           | Базовый класс: форматирование, состояние ошибок, статус |
| `basic_istream`  | `istream`       | Поток ввода (чтение) |
| `basic_ostream`  | `ostream`       | Поток вывода (запись) |
| `basic_iostream` | `iostream`      | Поток ввода/вывода |
| `basic_ifstream` | `ifstream`      | Файловый ввод |
| `basic_ofstream` | `ofstream`      | Файловый вывод |
| `basic_fstream`  | `fstream`       | Файловый ввод/вывод |

> 🧠 Важно: `cin` – объект класса `istream`, `cout` – объект класса `ostream`.

## Перегрузка операций `<<` и `>>` для своих классов

### Почему нельзя сделать оператор методом класса?

Выражение `cout << obj` имеет **левый операнд** – поток (`cout`), а **правый** – ваш объект.

Если бы оператор был **методом**, он вызывался бы у левого операнда:  
`cout.operator<<(obj)`.  
Но класс `ostream` не знает о вашем классе, и вы не можете добавить в него новый метод.

**Решение:** операторы должны быть **глобальными (внешними) функциями**, принимающими два аргумента: поток и объект.

---

### Общий шаблон для оператора вывода (`<<`)

```cpp
ostream& operator<<(ostream& stream, const YourClass& obj) {
    // выводим поля obj в stream
    stream << ...;
    return stream;   // обязательно возвращаем поток
}
```

- Возвращаемое значение – **ссылка на `ostream`** – позволяет делать цепочки: `cout << a << b;`.
- Второй параметр – чаще всего **константная ссылка**, чтобы избежать копирования и разрешить доступ только для чтения.

---

### Общий шаблон для оператора ввода (`>>`)

```cpp
istream& operator>>(istream& stream, YourClass& obj) {
    // читаем из stream в поля obj
    stream >> obj.field1 >> obj.field2;
    return stream;   // обязательно возвращаем поток
}
```

- Второй параметр – **неконстантная ссылка**, потому что мы будем изменять объект.
- Возвращаемое значение – **ссылка на `istream`** – для цепочек `cin >> a >> b;`.

---

### Доступ к приватным полям

Глобальные функции не имеют доступа к `private`-членам класса. Чтобы это разрешить, объявите их **дружественными** (friend) внутри класса:

```cpp
class YourClass {
private:
    int data;
public:
    // ...
    friend ostream& operator<<(ostream&, const YourClass&);
    friend istream& operator>>(istream&, YourClass&);
};
```

---

### Полный пример для класса `ThreeD`

```cpp
#include <iostream>
using namespace std;

class ThreeD {
private:
    int x, y, z;
public:
    ThreeD(int a, int b, int c) : x(a), y(b), z(c) {}

    // Объявляем друзей
    friend ostream& operator<<(ostream&, const ThreeD&);
    friend istream& operator>>(istream&, ThreeD&);
};

// Оператор вывода
ostream& operator<<(ostream& stream, const ThreeD& obj) {
    stream << obj.x << ", " << obj.y << ", " << obj.z;
    return stream;
}

// Оператор ввода
istream& operator>>(istream& stream, ThreeD& obj) {
    cout << "Enter X, Y, Z: ";
    stream >> obj.x >> obj.y >> obj.z;
    return stream;
}

int main() {
    ThreeD a(1, 2, 3);
    cout << a << endl;   // вывод
    cin >> a;            // ввод
    cout << a << endl;
    return 0;
}
```

## Флаги форматирования 

Каждый поток (например, `cout`) хранит в себе набор **флагов** — это битовые маски, каждая из которых включает или выключает какую-то опцию вывода. Эти флаги определены в классе `ios_base` (предке `ios`) в виде перечисления `fmtflags`.

Вот полный список основных флагов, разделённых по группам:

### Группа `basefield` (система счисления)
- **`dec`** — десятичная (по умолчанию).
- **`hex`** — шестнадцатеричная.
- **`oct`** — восьмеричная.
Эти три флага взаимоисключающие. Группа `basefield` используется для коллективной ссылки на них.

### Группа `adjustfield` (выравнивание)
- **`left`** — выравнивание по левому краю.
- **`right`** — по правому краю (по умолчанию).
- **`internal`** — выравнивание по знаку: знак выводится слева, а число прижимается к правому краю, а между ними — заполнители.

### Группа `floatfield` (формат чисел с плавающей точкой)
- **`fixed`** — фиксированный формат (без экспоненты).
- **`scientific`** — экспоненциальный (научный) формат.
Если не установлен ни один — компилятор выбирает сам (обычно `fixed` для малых чисел, `scientific` для очень больших).

### Одиночные флаги (не в группах)
- **`showpos`** — показывать знак `+` перед положительными числами.
- **`showbase`** — показывать основание системы счисления: для `hex` — `0x`, для `oct` — `0`.
- **`uppercase`** — использовать заглавные буквы в `hex` (например, `0X1F` вместо `0x1f`) и в экспоненте (`E` вместо `e`).
- **`showpoint`** — всегда показывать десятичную точку и хвостовые нули для вещественных чисел (даже если число целое, например, `5` выведется как `5.0`).
- **`boolalpha`** — выводить `true`/`false` вместо `1`/`0`.
- **`skipws`** — пропускать пробельные символы (пробелы, табуляции, переводы строк) при вводе из `cin` (по умолчанию включён).
- **`unitbuf`** — сбрасывать буфер после каждой операции вывода (используется редко, в основном для отладки).

## Управление флагами через функции-члены

### `setf()` — установка флагов
Синтаксис:
```cpp
fmtflags setf(fmtflags flags);
```
Она устанавливает переданные флаги и возвращает предыдущее состояние.

**Важный нюанс:** для флагов, входящих в группы (`basefield`, `adjustfield`, `floatfield`), нужно использовать **двухаргументную** версию `setf()`, иначе можно случайно включить несколько взаимоисключающих флагов одновременно (что вызовет неопределённое поведение). Двухаргументная версия:
```cpp
fmtflags setf(fmtflags flags, fmtflags mask);
```
Где `mask` — это группа, а `flags` — конкретный флаг внутри группы. Например, чтобы установить `hex`:
```cpp
cout.setf(ios::hex, ios::basefield); // правильно
// а просто cout.setf(ios::hex); — так делать не рекомендуется.
```
Однако, на практике многие используют упрощённую версию для одиночных флагов (`showpos`, `boolalpha`).

Пример:
```cpp
cout.setf(ios::showpos);          // включаем плюс перед положительными
cout.setf(ios::scientific);       // включаем научный формат
cout << 123.45;                   // выведет +1.234500e+002
```

Можно комбинировать через побитовое ИЛИ:
```cpp
cout.setf(ios::showpos | ios::scientific);
```

### `unsetf()` — сброс флагов
Синтаксис:
```cpp
void unsetf(fmtflags flags);
```
Сбрасывает указанные флаги. Остальные не трогает.
```cpp
cout.unsetf(ios::showpos); // убираем плюс
```

### `flags()` — получение/установка всех флагов разом
Без аргументов:
```cpp
fmtflags flags();
```
Возвращает текущую битовую маску всех флагов.

С аргументом:
```cpp
fmtflags flags(fmtflags f);
```
Устанавливает все флаги в соответствии с `f` (заменяя полностью) и возвращает предыдущие.

Пример проверки флага:
```cpp
if (cout.flags() & ios::showpos) {
    cout << "showpos включён\n";
}
```

## Управление шириной поля, точностью и заполнителем

### `width()` — минимальная ширина поля
Прототип:
```cpp
streamsize width(streamsize w);
```
Устанавливает минимальную ширину поля и возвращает предыдущую. Если следующее выводимое значение короче, чем `w`, оно дополняется **заполнителем** (по умолчанию пробел). Если длиннее — поле расширяется (значение не усекается).

**Критично:** `width()` действует **только на одну следующую операцию вывода**! После каждого вывода ширина сбрасывается в 0 (т.е. в значение по умолчанию). Поэтому её приходится вызывать перед каждым элементом, который нужно отформатировать.

Пример:
```cpp
cout.width(10);
cout << 42;    // выведет "        42" (8 пробелов + 2 цифры)
cout << 100;   // выведет "100" без отступов, так как width не установлен
```

### `precision()` — точность (количество цифр после запятой)
Прототип:
```cpp
streamsize precision(streamsize p);
```
Устанавливает точность и возвращает старую. По умолчанию 6.

**Важно:** точность работает по-разному в зависимости от формата:
    fixed печатает число с фиксированной точкой (например, 123.46).
    scientific печатает в экспоненциальной форме (например, 1.23e+02).

В обоих случаях precision(2) даст две цифры после запятой:
    123.456 в fixed → 123.46
    123.456 в scientific → 1.23e+02

Пример:
```cpp
cout.precision(2);
cout << 123.456; // в обычном режиме -> 1.2e+02 (общее знаков 2)
cout.setf(ios::fixed);
cout << 123.456; // теперь -> 123.46 (две после запятой)
```

Точность **действует постоянно**, пока вы её не переопределите.

### `fill()` — символ-заполнитель
Прототип:
```cpp
char fill(char ch);
```
Устанавливает символ, которым заполняются пустые места при использовании `width()`. Возвращает предыдущий символ. По умолчанию пробел.

Пример:
```cpp
cout.fill('*');
cout.width(10);
cout << 42; // выведет "********42"
```

## Что такое манипуляторы?

**Манипулятор** — это специальный объект или функция, которую можно вставить прямо в цепочку операций `<<` или `>>` для изменения состояния потока (форматирования, выравнивания, системы счисления и т.д.). Вместо вызова отдельных функций-членов (как `cout.setf()`) вы просто пишете `cout << hex << 255;` — это читается как «вывести 255 в шестнадцатеричном формате».

Манипуляторы бывают двух видов:
- **Без параметров** — например, `endl`, `hex`, `left`, `boolalpha`. Они определены в заголовке `<iostream>` (или `<ios>`).
- **С параметрами** — например, `setw(10)`, `setprecision(2)`, `setfill('*')`. Они определены в заголовке `<iomanip>` (манипуляторы ввода/вывода с параметрами). Этот заголовок нужно включать отдельно.

---

## Зачем нужны манипуляторы, если есть функции `setf()`/`width()`?

Главные преимущества:
1. **Читаемость** — код выглядит как декларация: «вывести в шестнадцатеричном виде с шириной 10».
2. **Группировка** — можно настроить несколько параметров в одной строке.
3. **Возможность создавать свои** — если часто повторяете набор настроек, можно сделать свой манипулятор.

---

## Стандартные манипуляторы

### Группа систем счисления (basefield)
- **`dec`** — десятичная система (по умолчанию).
- **`hex`** — шестнадцатеричная.
- **`oct`** — восьмеричная.
- **`setbase(int base)`** — устанавливает основание 8, 10 или 16 (принимает только эти значения; если передать другое, поведение не определено).

Эти манипуляторы влияют на все последующие целочисленные значения, пока не поменяете систему счисления.

### Группа выравнивания (adjustfield)
- **`left`** — выравнивание по левому краю.
- **`right`** — по правому (по умолчанию).
- **`internal`** — выравнивание по знаку: знак выводится слева, число — справа, а между ними вставляются заполнители.

Они работают в сочетании с `setw()` (ширина поля).

### Группа формата чисел с плавающей точкой (floatfield)
- **`fixed`** — фиксированный формат (без экспоненты).
- **`scientific`** — экспоненциальный (научный) формат.
- **`defaultfloat`** (C++11) — возвращает к стандартному (компилятор сам выбирает).
- **`hexfloat`** (C++11) — вывод в шестнадцатеричном научном формате (редко используется).

### Флаги отображения (boolean, знак, регистр)
- **`boolalpha`** — выводит `true`/`false` вместо 1/0.  
  `noboolalpha` — отключает (возвращает 1/0).
- **`showpos`** — показывает `+` перед положительными числами.  
  `noshowpos` — отключает.
- **`showbase`** — показывает основание системы счисления (например, `0x` для hex, `0` для oct).  
  `noshowbase` — отключает.
- **`uppercase`** — выводит буквы в hex и `E` в экспоненте заглавными.  
  `nouppercase` — строчными.
- **`showpoint`** — всегда выводит десятичную точку и хвостовые нули для вещественных чисел.  
  `noshowpoint` — отключает (по умолчанию точка выводится только при необходимости).

### Управление буфером и пробелами
- **`endl`** — вставляет символ новой строки (`\n`) и **сбрасывает** буфер (вызывает `flush`). Полезно для немедленного вывода.
- **`ends`** — вставляет нулевой символ (`'\0'`) (редко используется).
- **`flush`** — просто сбрасывает буфер без добавления перевода строки.
- **`unitbuf`** — включает режим сброса буфера после каждой операции вывода.  
  `nounitbuf` — отключает.
- **`skipws`** — при вводе пропускать ведущие пробельные символы (по умолчанию включено).  
  `noskipws` — не пропускать.

### Манипуляторы для работы с шириной, точностью, заполнителем
- **`setw(int w)`** — устанавливает минимальную ширину поля. **Действует только на следующую операцию вывода!** После вывода сбрасывается в 0.
- **`setprecision(int p)`** — устанавливает количество значащих цифр (или после запятой, в зависимости от `fixed`/`scientific`). **Действует постоянно**, пока не поменяете.
- **`setfill(char ch)`** — устанавливает символ-заполнитель для пустого места в поле. **Действует постоянно**.

### Универсальные манипуляторы для установки/сброса флагов
- **`setiosflags(fmtflags f)`** — устанавливает указанные флаги форматирования (например, `setiosflags(ios::showpos | ios::scientific)`).
- **`resetiosflags(fmtflags f)`** — сбрасывает указанные флаги.

### Вспомогательные манипуляторы
- **`ws`** — при вводе пропускает все ведущие пробельные символы (включая пробелы, табуляции, переводы строк). Часто используется перед чтением строки, чтобы очистить буфер от мусора.
- **`quoted`** (C++11) — позволяет вводить/выводить строки с пробелами в кавычках (например, `cin >> quoted(str)` прочитает `"Hello world"` как одну строку без кавычек).
- **`get_money` / `put_money`** — для работы с денежными суммами (локаль).
- **`get_time` / `put_time`** — для работы с датой/временем.

## Примеры использования стандартных манипуляторов

### Простой пример с `setprecision`, `setw`, `endl`
```cpp
#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    cout << setprecision(2) << 1000.243 << endl; // выведет 1e+003 (две значащие цифры)
    cout << setw(20) << "Hello there." << endl;  // выведет "    Hello there." (отступ 20)
    return 0;
}
```
*Обратите внимание:* `endl` используется без скобок, потому что это функция без параметров.

### Установка нескольких флагов через `setiosflags`
```cpp
cout << setiosflags(ios::showpos | ios::scientific) 
     << 123 << " " << 123.23 << endl;
```
Выведет: `+123 +1.232300e+002`

### Пропуск пробелов при вводе с `ws`
```cpp
char s[80];
cin >> ws >> s;   // сначала пропускаются все пробелы/табуляции/переводы строк, затем читается слово
cout << s;
```
Если пользователь нажмёт несколько раз Enter перед вводом, `ws` их пропустит.

### Использование `boolalpha`
```cpp
cout << boolalpha << true << " " << false << endl; // выведет "true false"
```

## Создание пользовательских манипуляторов

Вы можете написать свой манипулятор, чтобы сгруппировать часто используемые настройки или выполнить нестандартные действия.

### Манипуляторы вывода (для `ostream`)

**Сигнатура:**  
`ostream &имя(ostream &stream) { ... return stream; }`

Манипулятор принимает ссылку на поток, модифицирует его и возвращает ту же ссылку. При использовании в цепочке `cout << manip` компилятор вызывает эту функцию, передавая `cout` в качестве аргумента.

**Пример:** манипулятор `setup`, который выравнивает по левому краю, ставит ширину 10 и заполнитель `$`:
```cpp
#include <iostream>
#include <iomanip>
using namespace std;

ostream &setup(ostream &stream) {
    stream.setf(ios::left);          // левое выравнивание
    stream << setw(10) << setfill('$'); // ширина и заполнитель
    return stream;
}

int main() {
    cout << 10 << " " << setup << 10 << endl;
    // Вывод: 10 $$$$$$$10
    // Сначала выводится 10 (без форматирования), потом пробел,
    // затем манипулятор настраивает поток, и следующий 10 выводится с шириной 10,
    // выравниванием влево и заполнением '$'.
    return 0;
}
```
Здесь важно, что `setw` действует только на следующий вывод, поэтому мы вставили его в манипулятор, чтобы применять к каждому следующему числу (если бы мы хотели, чтобы он действовал каждый раз, пришлось бы переустанавливать его вручную перед каждым значением).

### Манипуляторы ввода (для `istream`)

**Сигнатура:**  
`istream &имя(istream &stream) { ... return stream; }`

**Пример:** манипулятор `prompt`, который переключает ввод в шестнадцатеричный режим и выводит приглашение:
```cpp
#include <iostream>
#include <iomanip>
using namespace std;

istream &prompt(istream &stream) {
    stream >> hex;   // устанавливаем чтение в hex
    cout << "Enter number in hex: ";
    return stream;
}

int main() {
    int i;
    cin >> prompt >> i;   // сначала применяется манипулятор, затем читается число
    cout << i << endl;    // выведет в десятичном виде (потому что hex действовал только на ввод)
    return 0;
}
```
Если пользователь введёт `1F`, переменная `i` получит значение 31, и выведется `31`.

---

## Важные нюансы при создании манипуляторов

1. **Возврат потока обязателен** — иначе цепочка `<<` сломается.
2. Манипулятор **не должен принимать аргументы**, если вы не используете сложные техники (создание параметризованных манипуляторов выходит за рамки книги, но в C++11+ это можно сделать через классы).
3. Манипуляторы можно комбинировать: `cout << left << setw(10) << 42;`.
4. Манипуляторы без параметров (например, `endl`) определены в пространстве имён `std` и не требуют `<iomanip>`.

## Основы: подключение и типы потоков

Для работы с файлами в C++ необходимо подключить заголовочный файл `<fstream>`. В нём определены три основных класса для работы с файловыми потоками:

*   **`ifstream`** (input file stream) — для чтения данных из файла.
*   **`ofstream`** (output file stream) — для записи данных в файл.
*   **`fstream`** (file stream) — для чтения и записи одновременно.

```cpp
#include <fstream>
using namespace std;

ifstream in;   // поток для ввода
ofstream out;  // поток для вывода
fstream both;  // поток для ввода/вывода
```

---

## Открытие файла: два способа

### Способ 1: через конструктор (наиболее распространённый)
Классы `ifstream`, `ofstream` и `fstream` имеют конструкторы, которые сразу открывают файл. Параметры конструктора совпадают с параметрами функции `open()`.

```cpp
ifstream in("myfile.txt");        // открыть для чтения
ofstream out("myfile.txt");       // открыть для записи (старое содержимое удаляется)
fstream both("myfile.txt");       // открыть для чтения и записи
```

### Способ 2: через функцию `open()`
Сначала создаётся объект потока, затем вызывается метод `open()`.

```cpp
ifstream in;
in.open("myfile.txt");
```

**Прототипы `open()`:**
```cpp
void ifstream::open(const char *имя_файла, ios::openmode режим = ios::in);
void ofstream::open(const char *имя_файла, ios::openmode режим = ios::out | ios::trunc);
void fstream::open(const char *имя_файла, ios::openmode режим = ios::in | ios::out);
```

---

## Режимы открытия файла (`ios::openmode`)

Режимы задаются с помощью перечисления `ios::openmode`. Их можно комбинировать через побитовое ИЛИ (`|`).

| Режим | Описание |
|-------|----------|
| `ios::in` | Открыть для чтения. |
| `ios::out` | Открыть для записи. |
| `ios::app` | (append) Все записи добавляются в конец файла. Только для вывода. |
| `ios::ate` | (at end) При открытии переместить указатель в конец файла, но чтение/запись возможны в любом месте. |
| `ios::trunc` | (truncate) Если файл существует, его содержимое удаляется (усекается до 0). |
| `ios::binary` | Открыть файл в двоичном режиме (без преобразования символов). По умолчанию — текстовый режим. |

**Важно:** для `ofstream` режим `ios::trunc` действует по умолчанию, поэтому старый файл при открытии затирается. Если нужно дозаписывать в конец, используйте `ios::app`.

**Примеры:**
```cpp
ofstream out("file.txt", ios::app);                 // дозапись в конец
ifstream in("file.bin", ios::in | ios::binary);     // двоичное чтение
fstream both("file.txt", ios::in | ios::out);       // чтение и запись
```

---

## Проверка успешности открытия

Файл может не открыться (нет прав, файл не существует и т.д.). Всегда проверяйте результат!

**Способ 1: через оператор `!` (поток приводится к `bool`)**
```cpp
if (!in) {
    cerr << "Не удалось открыть файл!\n";
    return 1;
}
```

**Способ 2: через метод `is_open()`**
```cpp
if (!in.is_open()) {
    cerr << "Файл не открыт!\n";
    return 1;
}
```
`is_open()` возвращает `true`, если поток связан с открытым файлом.

## Закрытие файла

Файл закрывается методом `close()`. Это освобождает ресурсы и сбрасывает буферы. Хотя файл закроется автоматически при уничтожении объекта, явный вызов `close()` — хорошая практика.

```cpp
in.close();
out.close();
```

## Чтение и запись текстовых файлов (форматированный ввод/вывод)

Для текстовых файлов проще всего использовать операции `<<` и `>>` — они работают так же, как с `cin` и `cout`.

### Запись в текстовый файл
```cpp
ofstream out("test.txt");
if (!out) { /* ошибка */ }

out << 10 << " " << 123.23 << "\n";
out << "This is a short text file.";
out.close();
```

### Чтение из текстового файла
```cpp
ifstream in("test.txt");
if (!in) { /* ошибка */ }

int i;
float f;
char ch;
char str[80];

in >> i;      // читаем 10
in >> f;      // читаем 123.23
in >> ch;     // читаем один символ (пропустит пробел)
in >> str;    // читает слово до пробела

cout << i << " " << f << " " << ch << "\n";
cout << str;
```

**Важные нюансы:**
- Оператор `>>` пропускает пробельные символы (пробелы, табуляции, переводы строк).
- При чтении строки (`char*` или `string`) `>>` останавливается на первом же пробеле. Чтобы прочитать строку с пробелами, используйте `getline()` (см. ниже).
- Для чтения и записи в двоичном режиме (без преобразований) нужно указать `ios::binary`.

---

## Чтение и запись в двоичном режиме (неформатированный ввод/вывод)

Двоичный режим полезен, когда:
- Нужно сохранить данные в том виде, в котором они лежат в памяти (массивы, структуры).
- Важна производительность (нет затрат на преобразование в текст).
- Не нужно, чтобы символы преобразовывались (например, `\r\n` -> `\n`).

Чтобы открыть файл в двоичном режиме, добавляем `ios::binary`.

### Поcимвольный ввод/вывод: `get()` и `put()`

- `put(char ch)` — записывает один байт (символ) в файл.
- `get(char &ch)` — читает один байт из файла в переменную `ch`.

```cpp
// Запись строки в двоичном режиме (без преобразований)
char *p = "hello there\n";
ofstream out("test", ios::out | ios::binary);
while (*p) out.put(*p++);
out.close();

// Чтение и вывод содержимого файла посимвольно
ifstream in("test", ios::in | ios::binary);
char ch;
while (in.get(ch)) {   // когда достигнут EOF, in станет false
    cout << ch;
}
in.close();
```

### Блочный ввод/вывод: `read()` и `write()`

Эти функции читают/записывают сразу блоки памяти заданного размера.

```cpp
istream &read(char *buf, streamsize num);
ostream &write(const char *buf, streamsize num);
```

- `read()` — читает `num` байт из файла в буфер `buf`.
- `write()` — записывает `num` байт из буфера `buf` в файл.

**Обязательно приводите указатель к `char*`**, даже если у вас массив другого типа.

```cpp
int numbers[5] = {1, 2, 3, 4, 5};

// Запись массива в файл
ofstream out("data.bin", ios::out | ios::binary);
out.write((char*)numbers, sizeof(numbers));
out.close();

// Очищаем массив и читаем обратно
int arr[5] = {0};
ifstream in("data.bin", ios::in | ios::binary);
in.read((char*)arr, sizeof(arr));
in.close();

for (int i = 0; i < 5; i++) cout << arr[i] << " "; // 1 2 3 4 5
```

**Важно:** если при чтении `read()` достигнет конца файла раньше, чем прочитает `num` байт, то в буфере будет только то, что успели прочитать. Узнать точное количество прочитанных байт можно с помощью метода `gcount()`:
```cpp
streamsize count = in.gcount(); // сколько байт было прочитано последним вызовом read()
```

## Дополнительные полезные функции

### Различные версии `get()`

Существуют перегруженные версии `get()` для чтения строк:

```cpp
istream &get(char *buf, streamsize num);               // читает до num-1 символов или до '\n'
istream &get(char *buf, streamsize num, char delim);   // читает до num-1 или до delim
int get();                                             // читает один символ (возвращает int)
```

**Важное отличие от `getline()`:** `get()` **не извлекает** разделитель (новую строку или `delim`) из потока. Он остаётся там, и следующий вызов `get()` его увидит.

```cpp
char str[80];
cin.get(str, 79);   // читает до 79 символов или до '\n'
// Если ввели "Hello World", str будет "Hello World", но '\n' останется в буфере.
```

### `getline()`

Очень похожа на `get()`, но **извлекает и удаляет** разделитель из потока.

```cpp
istream &getline(char *buf, streamsize num);
istream &getline(char *buf, streamsize num, char delim);
```

Пример:
```cpp
char str[80];
cin.getline(str, 79);  // читает всю строку вместе с пробелами, '\n' удаляется
```

**Разница `get()` vs `getline()`**:

| Функция | Что делает с разделителем |
|---------|---------------------------|
| `get(buf, num)` | Оставляет разделитель в потоке. |
| `getline(buf, num)` | Извлекает и удаляет разделитель. |

### `eof()` — проверка конца файла

```cpp
bool eof();
```
Возвращает `true`, если достигнут конец файла.

Пример:
```cpp
ifstream in("file.txt");
char ch;
while (!in.eof()) {
    in.get(ch);
    // обработка ch
}
```
**Лучше использовать проверку после чтения**, как в примере с `while (in.get(ch))`, потому что `eof()` становится `true` только после неудачной попытки чтения.

### `peek()` — посмотреть следующий символ без извлечения

```cpp
int peek();
```
Возвращает следующий символ в потоке или `EOF`, не удаляя его.

### `putback()` — вернуть символ обратно в поток

```cpp
istream &putback(char c);
```
Позволяет "откатить" последний считанный символ.

### `flush()` — принудительная запись буфера

```cpp
ostream &flush();
```
Заставляет немедленно записать все данные из буфера на диск. Полезно в критических ситуациях (например, при частых сбоях питания). Обычно буфер сбрасывается автоматически при закрытии файла или завершении программы.

## Произвольный (прямой) доступ к файлу

До сих пор мы читали и писали файлы последовательно — каждый следующий байт шёл сразу за предыдущим. Но часто нужно «перепрыгнуть» в нужное место, например, в базу данных или в большой бинарный файл. Для этого в C++ есть специальные функции, которые перемещают **указатели чтения** и **указатели записи** внутри файла.

### Указатели чтения и записи

У каждого открытого файлового потока есть два внутренних указателя:
- **Указатель чтения** (get pointer) — определяет, откуда будет считан следующий байт при вызове `read()`, `get()` и т.д.
- **Указатель записи** (put pointer) — определяет, куда будет записан следующий байт при вызове `write()`, `put()` и т.д.

Для потоков `ifstream` доступен только указатель чтения, для `ofstream` — только записи, а для `fstream` — оба.

### Функции перемещения: `seekg()` и `seekp()`

- **`seekg()`** (seek get) — перемещает указатель чтения.
- **`seekp()`** (seek put) — перемещает указатель записи.

Их основные прототипы:

```cpp
istream &seekg(off_type offset, seekdir origin);
ostream &seekp(off_type offset, seekdir origin);
```

- `offset` — смещение в байтах (целое число, может быть отрицательным).
- `origin` — точка отсчёта. Это перечисление `seekdir` с тремя значениями:

| Константа      | Описание                          |
|----------------|-----------------------------------|
| `ios::beg`     | от начала файла                   |
| `ios::cur`     | от текущей позиции указателя      |
| `ios::end`     | от конца файла                    |

`off_type` — это целочисленный тип (обычно `long long`), способный хранить любые смещения в файле.

**Примеры перемещения:**
```cpp
in.seekg(10, ios::beg);   // перейти к 10-му байту от начала файла
in.seekg(-5, ios::cur);   // отступить на 5 байт назад от текущей позиции
in.seekg(0, ios::end);    // перейти в конец файла (например, для определения размера)
out.seekp(100, ios::beg); // установить указатель записи на 100-й байт
```

Также существуют перегруженные версии с одним аргументом:
```cpp
istream &seekg(pos_type position);
ostream &seekp(pos_type position);
```
Они перемещают указатель в абсолютную позицию, которую обычно получают через `tellg()`/`tellp()`.

### Определение текущей позиции: `tellg()` и `tellp()`

- `tellg()` возвращает текущую позицию указателя чтения.
- `tellp()` возвращает текущую позицию указателя записи.

Возвращаемый тип — `pos_type` (аналог `off_type`, но для позиций).

```cpp
streampos pos = in.tellg();   // запомнить позицию
// ...做些操作...
in.seekg(pos);                // вернуться назад
```

### Важное предостережение: используйте двоичный режим!

Произвольный доступ **рекомендуется выполнять только для файлов, открытых в двоичном режиме** (`ios::binary`). Почему?

В текстовом режиме некоторые символы могут преобразовываться (например, `\r\n` в Windows превращается в `\n` при чтении и обратно при записи). Из-за этого реальная длина файла в байтах может не совпадать с тем, что вы видите в потоке, и смещения, рассчитанные по байтам, приведут к неверной позиции. В двоичном режиме таких преобразований нет, и байт в файле соответствует байту в потоке «один к одному».

---

### Практические примеры

**Пример 1: изменение байта в файле по номеру**
Программа принимает имя файла и номер байта (считая от 0), после чего записывает в эту позицию символ `'X'`.

```cpp
#include <iostream>
#include <fstream>
#include <cstdlib>
using namespace std;

int main(int argc, char *argv[]) {
    if (argc != 3) {
        cout << "Usage: CHANGE <filename> <byte>\n";
        return 1;
    }

    fstream out(argv[1], ios::in | ios::out | ios::binary);
    if (!out) {
        cout << "Cannot open file.\n";
        return 1;
    }

    long pos = atol(argv[2]);   // преобразуем строку в число
    out.seekp(pos, ios::beg);   // перемещаем указатель записи
    out.put('X');               // записываем байт
    out.close();
    return 0;
}
```

**Пример 2: вывод содержимого файла, начиная с заданной позиции**
```cpp
#include <iostream>
#include <fstream>
#include <cstdlib>
using namespace std;

int main(int argc, char *argv[]) {
    if (argc != 3) {
        cout << "Usage: SHOW <filename> <start>\n";
        return 1;
    }

    ifstream in(argv[1], ios::in | ios::binary);
    if (!in) {
        cout << "Cannot open file.\n";
        return 1;
    }

    in.seekg(atol(argv[2]), ios::beg);   // перемещаем указатель чтения
    char ch;
    while (in.get(ch))
        cout << ch;
    in.close();
    return 0;
}
```

## Состояние потока и обработка ошибок

Каждая операция ввода/вывода может завершиться успешно или с ошибкой. В C++ состояние каждого потока хранится в специальном объекте — флагах состояния. Это позволяет точно определить, что пошло не так.

### Флаги состояния (битовые маски)

Тип `iostate` (определён в `ios`) — это перечисление, но на практике это битовые флаги, которые можно комбинировать. Вот их значения:

| Флаг           | Описание                                                                 |
|----------------|--------------------------------------------------------------------------|
| `ios::goodbit` | 0 — все хорошо, ошибок нет.                                              |
| `ios::eofbit`  | 1, если достигнут конец файла (EOF).                                     |
| `ios::failbit` | 1, если произошла **восстановимая** ошибка (например, ожидалось число, а пришла буква). Поток остаётся работоспособным, но чтение/запись блокируется до сброса флага. |
| `ios::badbit`  | 1, если произошла **неисправимая** ошибка (например, потеря данных, повреждение потока). Поток становится бесполезным. |

Обычно `failbit` и `badbit` вместе означают, что операция не удалась, но природа ошибки разная.

### Функции для проверки состояния

Все они — члены класса `ios`, доступны для любого потока (`cin`, `cout`, файловых и т.д.):

- **`bool good() const`** — возвращает `true`, если ни один флаг (`eofbit`, `failbit`, `badbit`) не установлен, т.е. поток в нормальном состоянии.
- **`bool eof() const`** — возвращает `true`, если установлен `eofbit`.
- **`bool fail() const`** — возвращает `true`, если установлен `failbit` или `badbit`.
- **`bool bad() const`** — возвращает `true`, если установлен `badbit`.

**Важно:** `fail()` вернёт `true` и при `badbit`, и при `failbit`, а `bad()` — только при `badbit`. Поэтому часто проверяют `fail()`, чтобы узнать, была ли ошибка (включая конец файла, который устанавливает `eofbit`, но `fail()` обычно не срабатывает на `eofbit` — только если чтение пыталось за пределами файла).

**Пример:**
```cpp
ifstream in("data.txt");
int n;
in >> n;
if (in.fail()) {
    // не удалось прочитать число (либо конец файла, либо нечисловой формат)
}
```

### Получение полного состояния: `rdstate()`

```cpp
ios::iostate rdstate() const;
```
Возвращает битовую маску текущего состояния. Можно сравнивать с отдельными флагами:

```cpp
if (in.rdstate() & ios::eofbit) {
    cout << "Достигнут конец файла\n";
}
```

### Сброс состояния: `clear()`

```cpp
void clear(ios::iostate state = ios::goodbit);
```
- Если вызвать без аргументов (`clear()`) — сбрасывает все флаги, приводя поток в состояние `good`.
- Если передать конкретные флаги, они устанавливаются (обычно используют для принудительной установки ошибок, но чаще просто вызывают `clear()` для восстановления после ошибки).

**Пример:** после неудачного чтения вы можете очистить ошибку и попробовать снова (например, пропустить неверный ввод).
```cpp
int value;
cin >> value;
if (cin.fail()) {
    cin.clear();           // сброс ошибки
    cin.ignore(100, '\n'); // пропустить остаток строки
}
```

### Когда проверять состояние?

Хорошая практика — проверять состояние **после каждой операции ввода/вывода**, особенно при работе с файлами. В предыдущих примерах мы использовали `if (!out)`, что эквивалентно `if (out.fail())`. Это удобно, потому что объект потока неявно преобразуется в `bool` — он становится `true`, если `fail()` == `false`, и `false`, если `fail()` == `true`.

Таким образом, вы можете писать:
```cpp
if (!in) { ... }   // эквивалентно if (in.fail())
```

Для цикла чтения до конца файла безопаснее использовать:
```cpp
while (in.get(ch)) { ... }
```
Здесь `get()` возвращает ссылку на поток, а он преобразуется в `bool` — цикл продолжается, пока чтение успешно. При достижении EOF или ошибке `fail()` станет `true`, цикл завершится.
