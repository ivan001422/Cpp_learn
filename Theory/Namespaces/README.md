## Пространства имён (namespace)

**Проблема:** Конфликт имён (например, два класса `Stack` из разных библиотек).
**Решение:** Пространства имён создают "контейнеры" для идентификаторов, чтобы изолировать их друг от друга.

### Объявление
```cpp
namespace ИмяПространства {
    // Переменные, функции, классы
}
```

### Доступ к членам извне
Используется оператор `::` (разрешение области видимости):
```cpp
ИмяПространства::переменная = 10;
ИмяПространства::Класс объект;
```

### Пример (CounterNameSpace)
```cpp
namespace CounterNameSpace {
    int upperbound;
    class counter { /* ... */ };
}

// В main():
CounterNameSpace::upperbound = 100;
CounterNameSpace::counter ob1(10); // Объявление объекта
```

### Особенность
Пространства имён можно **дополнять** в разных частях программы или файлах:
```cpp
namespace NS { int i; }
// ... позже ...
namespace NS { int j; } // i и j теперь в одном NS
```

### Стандартное пространство `std`
Вся стандартная библиотека C++ находится в пространстве имён `std`. Поэтому мы пишем `using namespace std;` или явно `std::cout`.

### Инструкция `using`

**Проблема:** Постоянно писать `ИмяПространства::` перед каждым членом утомительно.
**Решение:** Инструкция `using` вводит члены пространства в текущую область видимости.

**Два формата:**

1.  **`using namespace Имя;`** — делает **все** члены пространства видимыми без квалификатора.
2.  **`using Имя::член;`** — делает видимым **только** указанный член.

```cpp
namespace CounterNameSpace {
    int upperbound;
    int lowerbound;
}

// В функции main():
using CounterNameSpace::upperbound; // теперь upperbound доступен напрямую
upperbound = 100; // OK

// lowerbound всё ещё требует квалификации:
CounterNameSpace::lowerbound = 0;

// Позже можно добавить всё пространство:
using namespace CounterNameSpace;
counter ob2(20); // теперь можно создавать объекты без префикса
```

**Важно:** Использование `using` не переопределяет другие пространства, а дополняет текущую область видимости.

### Неименованные пространства имён

**Суть:** Создают идентификаторы, уникальные для **одного файла** (аналог `static` на уровне файла, но более предпочтительный в C++).

**Синтаксис:**
```cpp
namespace {
    // объявления (переменные, функции, классы)
}
```
Члены такого пространства доступны напрямую внутри этого файла, но не видны из других файлов.

---

### Пространство имён `std`

Вся стандартная библиотека C++ находится в пространстве имён `std`.

- `using namespace std;` — делает все стандартные имена (cout, cin, vector и др.) доступными без префикса.
- Можно использовать префикс явно: `std::cout << "Hello";` — это хороший тон, если вы используете библиотеку редко, чтобы избежать загрязнения глобального пространства.
