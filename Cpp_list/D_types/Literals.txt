Литералы C++


Литералы - это фиксированные, предназначенные для чтения человеком значения, которые не могут быть изменены программой.
Также они называются константами.

Литералы в С++ могут иметь любой базовый тип данных. Способ представления каждого литерала зависит от его типа.
Символьные литералы заключаются в одинарные кавычки (например, 'а' и '%').

'A'        // char (обычный)
u'A'       // char16_t (UTF-16, C++11)
U'A'       // char32_t (UTF-32, C++11)
L'A'       // wchar_t (широкий символ)


Целочисленные литералы задаются как числа без дробной части (такие, как 18 и -100). Литералы с плавающей точкой содержат
десятичную точку, за которой следует дробная часть числа (к примеру, 11.123). Для констант с плавающей точкой можно также
использовать экспоненциальное представление чисел.

123      // int (по умолчанию)
123u     // unsigned int
123l     // long
123ll    // long long (C++11)
123ul    // unsigned long
123ull   // unsigned long long (C++11)

Каждое литеральное значение имеет какой-либо тип данных. Но как вы уже знаете, есть несколько разных целочисленных типов,
таких как int, short int и unsigned long int. Существует также три типа данных с плавающей точкой float, double и long double
. Как же компилятор определяет тип литерала? Нa- пример, число 123.23 имеет тип float или double? Ответ на этот вопрос состоит
из двух частей. Во-первых, компилятор С++ автоматически делает некоторые предположения насчет литералов. Во-вторых, при желании 
программист может явно указать тип литерала.

По умолчанию компилятор связывает целочисленный литерал с совместимым и одновременно наименьшим по занимаемой памяти типом данных,
начиная c int. Следовательно, для 16-битных сред число 10 будет связано с int, а 103000 - c long int. И хотя значение 10 можно
соотнести с char, компилятор этого не сделает, ведь это означает выход за пределы данного типа.

Единственным исключением из правила «наименьшего типа» являются константы с плавающей точкой, которым по умолчанию присваивается тип double.

Для начинающего программиста такие стандарты работы компилятора вполне приемлемы. Однако у вас есть возможность четко определить нужный тип.
Чтобы задать точный тип числовой константы, нужно выбрать соответствующий суффикс. Для типов с плавающей точкой действуют следующие суффиксы:
если число с плавающей точкой завершить буквой F, оно будет обрабатываться с использованием типа float, а если буквой L, подразумевается тип
long double. Для целочисленных типов суффикс U означает использование модификатора типа unsigned, а суффикс L - long.
(Для того чтобы задать модификатор unsigned long, укажите оба суффикса, U и L.) 

3.14f    // float
3.14     // double (по умолчанию)
3.14l    // long double

Литералы шестнадцатеричной и восьмеричной систем счисления.

Иногда удобнее использовать шестнадцатеричную или восьмеричную системы счисления.
Для этого в C++ разрешено задавать целочисленные литералы в этих системах.
Для восьмеричного литерала в начале используется префикс 0.
Для шестнадцатеричного литерала в начале используется префикс 0x или 0X.

hex = 0xFF // 255 в десятичной системе 
oct = 011 // 9 в десятичной системе






Управляющие символьные последовательности:

Хотя литералы и управляющие последовательности — различные языковые конструкции, они тесно связаны:
любая управляющая последовательность может присутствовать исключительно как составная часть 
символьного или строкового литерала.

С выводом большинства печатаемых символов прекрасно справляются символьные константы, заключенные
в одинарные кавычки. Однако некоторые из них (например, символ возврата каретки) невозможно
ввести в исходный код программы с клавиатуры. Часть символов (например, одинарные и двойные кавычки)
имеет специальное назначение в C++, поэтому иногда их нельзя ввести напрямую. По этой причине в языке
C++ разрешено использовать ряд специальных символьных последовательностей с обратным слешем, которые
также называются управляющими последовательностями (табл. 2.3).

Таблица 2.3. Управляющие символьные последовательности
Код	Значение
\b	Возврат на одну позицию (Backspace)
\f	Подача страницы (Form feed)
\n	Новая строка (New line)
\r	Возврат каретки (Carriage return)
\t	Горизонтальный отступ (Tab)
\"	Двойная кавычка
\'	Одинарная кавычка (апостроф)
\\	Обратный слеш
\v	Вертикальный отступ (Vertical tab)
\a	Звуковой сигнал (Alert/Bell)
\?	Вопросительный знак (используется в тройных трigraph-последовательностях)
\N	Восьмеричная константа (где N — восьмеричное число)
\xN	Шестнадцатеричная константа (где N — шестнадцатеричное число)

Пример использования:


#include <iostream>
using namespace std;

int main() {
    cout << "Привет,\nмир!\n";          // Перевод строки
    cout << "Табуляция:\tтекст\n";      // Горизонтальный отступ
    cout << "Кавычки: \"C++\"\n";       // Двойные кавычки внутри строки
    cout << "Звук:\a\n";                // Звуковой сигнал
    cout << "Обратный слеш: \\n\n";     // Вывод самого обратного слеша
    cout << "Восьмеричный: \101\n";     // Символ 'A' в ASCII (восьмеричный 101)
    cout << "Шестнадцатеричный: \x41\n"; // Символ 'A' в ASCII (шестнадцатеричный 41)
    return 0;
}

Примечание:
Управляющие последовательности начинаются с обратного слеша \ и интерпретируются компилятором как специальные символы, а не как обычный текст.
