# Модификаторы и квалификаторы в C++

В C++ существуют ключевые слова, которые изменяют свойства типов (`signed`, `unsigned`, `short`, `long`, `long long`) или поведение переменных и функций (`const`, `volatile`, `static`, `extern`, `auto`, `register`).  
Ниже приведены краткие описания каждого из них с примерами.

---

## 1. `signed` и `unsigned`

**Назначение:** Управляют представлением знака у целочисленных типов.  
По умолчанию целые типы являются знаковыми (`signed`).
```

---
| Ключевое слово | Диапазон (для 4-байтового `int`)   | Особенность                                      |
|----------------|------------------------------------|--------------------------------------------------|
| `signed`       | от –2 147 483 648 до 2 147 483 647 | Старший бит хранит знак (0 → +, 1 → −)           |
| `unsigned`     | от 0 до 4 294 967 295              | Все биты участвуют в значении, нет отрицательных |

```cpp
signed int a = -100;           // можно просто int a = -100;
unsigned int b = 3000000000U;  // U – литерал unsigned
unsigned char c = 255;         // максимум для 1 байта


---

## 2. `short`, `long`, `long long`

**Назначение:** Изменяют размер целочисленного типа (модификаторы длины).
```

---
| Модификатор | Типичный размер | Диапазон (знаковый)                               |
|-------------|-----------------|---------------------------------------------------|
| `short`     | 2 байта         | от –32 768 до 32 767                              |
| `long`      | 4 или 8 байт    | от –2 147 483 648 до 2 147 483 647 (4 байта)      |
| `long long` | не менее 8 байт | от –9.22·10¹⁸ до 9.22·10¹⁸                        |

```cpp
short s = 32000;
long l = 2147483647L;            // L – литерал long
long long ll = 9223372036854775807LL;  // LL – long long
unsigned long long ull = 18446744073709551615ULL;
```

---

## 3. `const`

**Назначение:** Объявляет константность — значение переменной нельзя изменить после инициализации.

- Для указателей:
  - `const int *p` – нельзя менять данные через указатель.
  - `int * const p` – нельзя менять адрес, но данные менять можно.
  - `const int * const p` – ни данные, ни адрес.

```cpp
const double PI = 3.14159;
int x = 10;
const int* ptr = &x;  // *ptr = 20; // ошибка
int* const cptr = &x; // cptr = &y; // ошибка
```

---

## 4. `volatile`

**Назначение:** Сообщает компилятору, что значение переменной может измениться в любой момент без видимого действия программы (аппаратно, из другого потока, обработчиком прерывания).  
Запрещает оптимизации, связанные с кешированием в регистрах — каждое чтение/запись идёт напрямую в память.

```cpp
volatile int sensor = 0;
int val = sensor;  // всегда читается из памяти, даже если только что прочитали
```

> **Примечание:** `volatile` не обеспечивает атомарность и не упорядочивает доступы к памяти (для этого нужны `std::atomic` или мьютексы).

---

## 5. `auto`

**Назначение:** Автоматический вывод типа переменной из инициализатора (начиная с C++11).

- Тип определяется во время компиляции.
- Переменная должна быть инициализирована сразу.
- Упрощает работу со сложными типами (итераторы, лямбды).

```cpp
auto x = 42;                     // int
auto pi = 3.14159;               // double
std::vector<int> v{1,2,3};
for (auto it = v.begin(); it != v.end(); ++it) { ... }
```

---

## 6. `extern`

**Назначение:** Указывает, что переменная или функция определена в другой единице трансляции (внешнее связывание).

- Не выделяет память, только объявляет.
- Позволяет использовать глобальные объекты из других `.cpp` файлов.
- `const`-глобалы по умолчанию имеют внутреннее связывание, `extern` делает их доступными извне.

```cpp
// file1.cpp
extern int global_counter;  // объявление
void print() { cout << global_counter; }

// file2.cpp
int global_counter = 100;   // определение
```

---

## 7. `static`

**Назначение:** В зависимости от контекста меняет **продолжительность хранения** и **связывание**.

1. **Локальная `static`** внутри функции:
   - Переменная создаётся один раз и сохраняет значение между вызовами.
   - Инициализируется при первом выполнении инструкции.
2. **Глобальная `static`** (переменная или функция на уровне файла):
   - Ограничивает видимость текущей единицей трансляции (внутреннее связывание).
   - Недоступна из других `.cpp` файлов.

```cpp
void counter() {
    static int count = 0;  // сохраняется между вызовами
    ++count;
}

static int hidden = 42;    // видна только в этом файле
```

---

## 8. `register`

**Назначение (историческое):** Подсказка компилятору хранить переменную в регистре процессора для ускорения доступа.

**Текущий статус:**
- В C++11 – **устаревшее** (deprecated).
- В **C++17** – удалено как спецификатор хранения, но **имя остаётся зарезервированным** (нельзя использовать `register` как идентификатор).
- Современные компиляторы игнорируют подсказку и сами распределяют регистры.
- Нельзя взять адрес `register`-переменной (исторически).

```cpp
// В C++14 и ранее:
register int sum = 0;   // подсказка (обычно игнорируется)
// int* p = &sum;       // ошибка: нельзя взять адрес

// В C++17 и новее:
// register int x;      // ОШИБКА: удалённый спецификатор
// int register;        // ОШИБКА: зарезервированное слово
```
