### Обработка исключительных ситуаций (Exceptions)

**Что это?** Исключение — это ошибка, возникающая во время выполнения программы (например, деление на ноль, отсутствие файла). Механизм исключений позволяет отделить код обработки ошибок от основного кода, делая программу чище и надёжнее.

#### Базовые элементы: `try`, `catch`, `throw`

В C++ обработка строится на трёх ключевых словах:

*   **`try`** — блок кода, который вы хотите проверить на наличие ошибок.
*   **`throw`** — используется для "выброса" ошибки (создания исключения).
*   **`catch`** — блок, который "ловит" выброшенное исключение и обрабатывает его.

**Схема работы:**
1.  Код, который может вызвать ошибку, помещается в блок `try`.
2.  При возникновении ошибки выполняется оператор `throw` с указанием типа и значения ошибки.
3.  Управление немедленно переходит к блоку `catch`, который соответствует типу выброшенного исключения.

**Практический пример (база):**

```cpp
#include <iostream>
using namespace std;

int main() {
    cout << "Start\n";
    try {
        cout << "Inside try block\n";
        throw 99; // Выбрасываем целочисленную ошибку
        cout << "Эта строка не выполнится\n";
    }
    catch (int i) { // Ловим исключение типа int
        cout << "Перехвачено исключение. Значение: " << i << "\n";
    }
    cout << "End\n";
    return 0;
}
```

**Важно:** После `throw` выполнение блока `try` прекращается, и управление переходит к `catch`. Тип в `catch` должен совпадать с типом в `throw`, иначе программа аварийно завершится.

#### Выброс исключения из функции

Исключение может быть выброшено не только в самом блоке `try`, но и внутри любой функции, вызванной из этого блока. Оно будет "поднято" вверх по стеку вызовов до ближайшего подходящего `catch`.

```cpp
void Xtest(int test) {
    if (test) throw test; // Исключение улетит в main()
}
int main() {
    try {
        Xtest(1); // Вызов функции, которая может бросить исключение
    } catch (int i) {
        cout << "Поймано в main: " << i;
    }
}
```

#### Множественные `catch` и иерархия классов

*   Можно использовать несколько блоков `catch` для обработки разных типов ошибок.
*   Если есть иерархия классов, `catch` для базового класса перехватит все исключения производных классов. Поэтому **обработчики производных классов должны идти перед обработчиками базовых классов**.

```cpp
catch (D d) { /* сначала дочерний */ } 
catch (B b) { /* потом родительский */ }
```

#### "Ловим всё" (`catch(...)`) и спецификации `throw`

*   **`catch(...)`** — перехватывает исключения любого типа. Используется как "последний рубеж" в конце цепочки `catch`.

```cpp
catch (...) { // Ловит всё, что не поймали предыдущие catch
    cout << "Неизвестная ошибка!";
}
```

*   **Спецификация `throw()`** (указывается после заголовка функции) ограничивает типы исключений, которые функция может выбрасывать наружу.

```cpp
// Функция может выбросить только int или char
void func(int test) throw(int, char) { ... }
```
> **Примечание:** Современные компиляторы (особенно MSVC) могут игнорировать эту спецификацию, относясь к ней как к документации. В новых стандартах C++ её использование не рекомендуется, вместо неё используют `noexcept`.

#### Повторное генерирование

Иногда в `catch` нужно выполнить часть действий (например, залогировать ошибку), а затем передать исключение выше(При вложенных try/catch). Для этого используется `throw` без аргументов.

```cpp
catch (char*) {
    // Сделать что-то локально
    throw; // Перебрасываем это же исключение дальше
}
```
