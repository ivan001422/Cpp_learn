# Перегрузка функций в C++

Перегрузка функций (function overloading) — это механизм, позволяющий объявить несколько функций с одним именем, но разными
параметрами. Компилятор сам выбирает нужную версию на основе переданных аргументов. Это один из способов реализации полиморфизма в C++.

## Как это работает

Достаточно объявить функции с одинаковыми именами, но различающимися `типами` и/или `количеством` параметров. Компилятор
анализирует аргументы при вызове и вызывает наиболее подходящую версию.

### Пример

```cpp
#include <iostream>
using namespace std;

// Прототипы перегруженных функций
int  del(int, int);
double del(double, double);

int main() {
    cout << del(10, 5) << '\n';    // вызовет int-версию
    cout << del(0.5, 0.1);         // вызовет double-версию
    return 0;
}

// Определения
int del(int x, int y) {
    return x / y;                  // целочисленное деление
}

double del(double x, double y) {
    return x / y;                  // деление с плавающей точкой
}
```

Результат:
```
2
5
```

## Правила перегрузки

- **Уникальность сигнатуры.** Каждая перегруженная функция должна иметь уникальную комбинацию типов параметров (их количество, порядок
- или сами типы).
- **Возвращаемый тип не учитывается.** Если две функции отличаются только типом возвращаемого значения, это не считается допустимой перегрузкой — компилятор не сможет однозначно выбрать версию.
- **Можно комбинировать** с другими механизмами (шаблонами, значениями по умолчанию), но осторожно, чтобы не возникло неоднозначности.

## Преимущества

- **Читаемость.** Вместо `delInt`, `delDouble` программист использует одно логическое имя `del`.
- **Удобство.** Код становится короче, а интерфейс функций — интуитивнее.
- **Расширяемость.** Можно добавлять новые версии для работы с новыми типами, не меняя существующий код вызова.

## Автоматическое преобразование типов и неоднозначность

При вызове перегруженных функций компилятор может выполнять стандартные преобразования типов аргументов (например, `short` → `int`, `float` → `double`). Если существует несколько подходящих версий и ни одна из них не является «лучшей» по правилам разрешения перегрузки, возникает **неоднозначность (ambiguity)** — ошибка компиляции.

### Пример безопасного преобразования

```cpp
void f(int x)   { cout << "int: " << x << '\n'; }
void f(double x){ cout << "double: " << x << '\n'; }

int i = 10;
double d = 10.1;
short s = 99;
float r = 11.5F;

f(i);   // int: 10
f(d);   // double: 10.1
f(s);   // int: 99      (short → int)
f(r);   // double: 11.5 (float → double)
```

Здесь для `short` и `float` выбирается наиболее подходящая версия без неоднозначности.

### Неоднозначный вызов

```cpp
void f(double x) { cout << "double\n"; }
void f(float x)  { cout << "float\n"; }

int t = 10;
f(10);   // ❌ ошибка: int можно привести и к float, и к double с одинаковым приоритетом
```

Аналогичная неоднозначность может возникнуть при различиях только в знаковости/размере целочисленных параметров (например, `unsigned int` и `int`).

## Аргументы по умолчанию

C++ позволяет задавать параметрам функции значения, которые используются автоматически, если при вызове соответствующий аргумент не передан. Это позволяет сократить количество перегруженных функций.

**Синтаксис:**

```cpp
void func(int a, bool b, double c = 3.14);
```

- Все параметры со значениями по умолчанию должны находиться **в конце списка** параметров.
- Можно задать несколько аргументов по умолчанию подряд.

### Пример

```cpp
void show(int x, int y = 20) {
    cout << x << " " << y << '\n';
}

show(5);       // 5 20
show(5, 99);   // 5 99
```

### Осторожность: неоднозначность с перегрузкой

Сочетание перегруженных функций и аргументов по умолчанию может привести к двусмысленности:

```cpp
int f(int i) {
    return i;
}
int f(int i, int j = 10) {
    return i * j;
}

f(4, 5);   // ok: вызовет f(int, int)
f(4);      // ❌ неоднозначность: вызвать f(int) или f(int, int) с j=10?
```

Компилятор не может выбрать, использовать ли одноаргументную версию или двухаргументную с умолчанием — программа не скомпилируется.

> **Рекомендация:** при проектировании перегруженных функций и параметров по умолчанию следите, чтобы не создавалось ситуаций, где один и тот же вызов может быть интерпретирован несколькими способами.
