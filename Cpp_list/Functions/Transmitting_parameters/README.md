# Два способа передачи аргументов в подпрограммы  
*(функции, методы, процедуры)*

В языках программирования, как правило, предусматривается **два способа** передачи аргументов в подпрограммы:

> 💡 **Важно:** По умолчанию в С++ используется **вызов по значению** (*call‑by‑value*).

---

### 1️⃣ Вызов по значению *(call‑by‑value)*

> При вызове по значению значение аргумента **копируется** в формальный параметр подпрограммы.

- Изменения, внесённые в параметр внутри подпрограммы, **не влияют** на исходный аргумент.
- Аргумент остаётся неизменным после вызова.

---

### 2️⃣ Вызов по ссылке *(call‑by‑reference)*

> При вызове по ссылке в формальный параметр копируется **адрес** аргумента (а не его значение).

- Параметр и аргумент **ссылаются на одну и ту же область памяти**.
- Изменения, внесённые в параметр, **напрямую влияют** на исходный аргумент.
-В языке C++ это реализуется через **указатели**.

---

### 📌 Ссылочные параметры в C++

-Ссылочные параметры (int &a) — это механизм языка C++, который позволяет:
-Передать объект в функцию без копирования (экономия памяти и времени)
-Изменять оригинал напрямую, без использования указателей
-Работать с синтаксисом, аналогичным передаче по значению
