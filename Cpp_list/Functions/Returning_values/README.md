## Возврат указателей и ссылок из функций

Функции могут возвращать не только значения, но и **указатели** или **ссылки**, предоставляя доступ к уже существующим данным. Это полезно для эффективной работы с памятью, но требует соблюдения правил безопасности.

### Возврат указателей

Функция объявляется с типом `тип*`. Она должна вернуть **действительный** адрес, который останется актуальным после выхода из функции.

```cpp
#include <iostream>
using namespace std;

int global = 42;

int* getGlobalPtr() {
    return &global;        // безопасно: возвращаем адрес глобальной переменной
}

int* createOnHeap() {
    int* ptr = new int(100);
    return ptr;            // безопасно: память выделена в куче
}

int main() {
    int* p = getGlobalPtr();
    cout << *p << endl;    // 42

    int* q = createOnHeap();
    cout << *q << endl;    // 100
    delete q;              // не забываем освободить память
    return 0;
}
```

**Ошибка!** Нельзя возвращать адрес локальной переменной:

```cpp
int* badPointer() {
    int local = 10;
    return &local;   // ❌ local уничтожится, указатель станет «висячим»
}
```

### Возврат ссылок

Функция объявляется с типом `тип&`. Она возвращает неявный указатель (ссылку) на объект, позволяя использовать результат вызова как сам объект — читать и изменять его.

```cpp
#include <iostream>
using namespace std;

double value = 3.14;

double& getRef() {
    return value;          // безопасно: ссылка на глобальную переменную
}

int main() {
    cout << getRef() << endl;   // 3.14
    getRef() = 2.71;            // меняем value через возвращённую ссылку
    cout << value << endl;      // 2.71
    return 0;
}
```

**Ошибка!** Нельзя возвращать ссылку на локальную переменную:

```cpp
int& badReference() {
    int local = 10;
    return local;   // ❌ local уничтожится, ссылка недействительна
}
```

### Общие правила безопасности

Безопасно возвращать указатель или ссылку на:

- **глобальные** и **статические** переменные;
- **динамически выделенную память** (через `new`);
- **параметры, переданные по указателю или ссылке**;
- **элементы контейнера**, который продолжает существовать (например, ссылка на поле класса).

```cpp
// Безопасный пример с параметром-указателем
int* getElementPtr(int* arr, int index) {
    return &arr[index];   // arr существует вне функции
}

// Безопасный пример с параметром-ссылкой
int& getElementRef(int* arr, int index) {
    return arr[index];    // arr существует вне функции
}
```

> **Кратко:** возврат указателей и ссылок даёт гибкость, но источник данных должен жить дольше, чем сама функция. В противном случае поведение программы неопределено.
