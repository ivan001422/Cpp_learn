# Приоритет операторов в C++

| Приоритет | Операторы                                                                 | Описание                                                       | Ассоциативность |
|-----------|---------------------------------------------------------------------------|----------------------------------------------------------------|-----------------|
| 1         | `::`                                                                      | Разрешение области видимости (scope resolution)                | слева направо   |
| 2         | `()` `[]` `->` `.`                                                        | Вызов функции, индексация, доступ к члену, доступ к члену (объект) | слева направо   |
|           | `++` `--` (постфиксные)                                                   | Постфиксный инкремент/декремент                                |                 |
|           | `typeid` `const_cast` `dynamic_cast` `static_cast` `reinterpret_cast`     | Идентификация типа, приведение типов                           |                 |
| 3         | `++` `--` (префиксные)                                                    | Префиксный инкремент/декремент                                 | **справа налево** |
|           | `+` `-` (унарные)                                                         | Унарный плюс/минус                                             |                 |
|           | `!` `~`                                                                   | Логическое НЕ, побитовое НЕ                                    |                 |
|           | `*` `&` (унарные)                                                         | Разыменование указателя, взятие адреса                         |                 |
|           | `sizeof`                                                                  | Размер типа/выражения                                          |                 |
|           | `new` `new[]` `delete` `delete[]`                                         | Динамическое выделение/освобождение памяти                     |                 |
|           | `(type)`                                                                  | Приведение типов в C-стиле                                     |                 |
| 4         | `.*` `->*`                                                                | Указатели на члены (member pointer access)                     | слева направо   |
| 5         | `*` `/` `%`                                                               | Умножение, деление, остаток от деления                         | слева направо   |
| 6         | `+` `-` (бинарные)                                                        | Сложение, вычитание                                            | слева направо   |
| 7         | `<<` `>>`                                                                 | Побитовый сдвиг влево/вправо                                   | слева направо   |
| 8         | `<` `<=` `>` `>=`                                                         | Операторы сравнения (меньше, меньше или равно, больше, больше или равно) | слева направо   |
| 9         | `==` `!=`                                                                 | Равно, не равно                                                | слева направо   |
| 10        | `&` (бинарный)                                                            | Побитовое И                                                    | слева направо   |
| 11        | `^`                                                                       | Побитовое исключающее ИЛИ (XOR)                                | слева направо   |
| 12        | `\|`                                                                      | Побитовое ИЛИ                                                  | слева направо   |
| 13        | `&&`                                                                      | Логическое И                                                   | слева направо   |
| 14        | `\|\|`                                                                    | Логическое ИЛИ                                                 | слева направо   |
| 15        | `? :`                                                                     | Тернарный условный оператор                                    | **справа налево** |
| 16        | `=` `+=` `-=` `*=` `/=` `%=` `<<=` `>>=` `&=` `^=` `\|=`                 | Присваивание и составные присваивания                          | **справа налево** |
| 17        | `,`                                                                       | Оператор запятая                                               | слева направо   |

## Примечания

- Группа 2 (постфиксные) и группа 3 (унарные) имеют разную ассоциативность: постфиксные читаются слева направо, унарные — справа налево. Это важно, например, в выражении `*p++` — сначала `p++` (постфиксный), затем `*` (унарный).
- Операторы приведения типов (`static_cast`, `dynamic_cast` и др.) и `typeid` находятся на втором уровне.
- В группе 16 перечислены основные составные присваивания; их ассоциативность — справа налево, что позволяет цепочки `a = b = c`.
- Тернарный оператор (15) также правоассоциативен, что позволяет вложенные конструкции `a ? b : c ? d : e`.
- Оператор запятая имеет самый низкий приоритет, поэтому выражения вроде `int x = 1, y = 2;` работают как ожидается.
