# Основы классов и объектов в C++

## Определение класса

Начнём с определения терминов класса и объекта.

**Класс** можно представить как некий шаблон, который определяет формат объекта. Он включает как данные (состояние | атрибуты), так и код (поведение | методы). В языке C++ спецификация класса используется для построения объектов.

**Объект** — это экземпляр класса.

Важно понимать, что класс — это логическая абстракция, которая реально не существует до тех пор, пока не будет создан объект этого класса (то, что станет физическим представлением класса в памяти компьютера).

Атрибуты в классе представляются в виде переменных, а методы — в виде функций. Эти функции и переменные называют **членами класса**.

Иногда вместо термина "атрибут класса" используется термин "переменная экземпляра".

---

## Синтаксис объявления класса

Класс создаётся с помощью ключевого слова `class`:

```cpp
class имя_класса {
    // приватные данные и функции (по умолчанию)
public:
    // публичные данные и функции
} список_объектов;
```

В список объектов можно сразу поместить объекты этого класса, но это необязательно. Экземпляры класса можно создавать и после его объявления прямо в коде.

Класс может содержать как приватные, так и публичные члены. По умолчанию все элементы, определённые в классе, являются **приватными** (ключевое слово `public` необязательно для перехода к публичной части, но само наличие `private` подразумевается по умолчанию). Следовательно, к приватным членам могут получить доступ только другие члены их класса, другие части программы этого сделать не могут.

Все члены класса, определённые после ключевого слова `public`, доступны для всех других частей программы. Как правило, в программе организуется доступ к приватным переменным через его публичные методы.

Возможность создания приватных членов класса является основой **инкапсуляции** (дословно — помещение в капсулу).

---

## Простейший пример создания класса с тремя атрибутами

```cpp
class Point {
public:
    float x;
    float y;
    float z;
};
```

*Напомним, что класс — это логическая абстракция, и на данный момент у нас нет ни одного реального объекта этого класса.*

---

## Создание объектов (экземпляров класса)

Синтаксис объявления переменной этого класса выглядит так:

```cpp
тип имя_переменной; // Ничем не отличается от обычного создания переменной с базовым типом
```

Дополним наш код при помощи данного синтаксиса:

```cpp
Point p1; // Создали экземпляр класса Point
```

Всякий раз при создании экземпляра класса создаётся объект, который содержит собственную копию каждой переменной экземпляра, определённой этим классом.

Для доступа к этим переменным воспользуемся **точечной нотацией** (`объект.член`). Напомним, что использование точечной нотации к члену класса доступно лишь тогда, когда описанный член (метод или атрибут) класса находится в публичном доступе.

Для нашего примера это может выглядеть так:

```cpp
p1.x = 1.0;
```

*Напомним, что каждый экземпляр класса имеет собственные копии переменных экземпляра, определённых в этом классе. Т.е. содержимое переменных в одном объекте может отличаться от содержимого аналогичных переменных в другом.*

## Добавление в класс функций-членов (методов)

Чтобы добавить функцию (метод) в класс, нужно добавить её прототип внутри объявления этого класса.

```cpp
class Point {
public:
    float x;
    float y;
    float z;

    void reset(); // Функция обнуления координат точки
};

int main() {
    Point p1;
}
```

Поскольку функции-члены обеспечены своими прототипами в определении класса, их не нужно помещать ни в какое другое место программы (имеется в виду, что прототипы уже объявлены в классе).

Чтобы реализовать функцию, которая является членом класса, сообщите компилятору, какому классу она принадлежит, квалифицировав имя функции именем класса. Вот как можно записать код функции `reset()`:

```cpp
void Point::reset() {
    x = 0;
    y = 0;
    z = 0;
}
```

Обратите внимание на операцию `::`, которая отделяет имя класса `Point` от имени функции `reset()`. Это **операция разрешения области видимости** (операция разрешения контекста). Она связывает имя класса с именем его члена, чтобы сообщить компилятору, какому классу принадлежит данная функция. Это очень важно, поскольку разные классы могут использовать одинаковые имена функций.

В теле функции `reset()` обращение к переменным происходит напрямую, то есть без использования точечной нотации. Если метод использует переменную экземпляра, которая определена в том же классе, то точечная нотация не нужна, поскольку метод всегда вызывается относительно некоторого объекта своего класса. Если вызов метода произошёл, значит, объект известен. Поэтому нет необходимости ещё раз указывать объект в функции-члене.

---

## Вызов методов

Методы можно вызывать только относительно заданного объекта. Это можно сделать двумя способами:

### 1) Вызов метода вне класса

Необходимо использовать точечную нотацию.

Пример:

```cpp
p1.reset(); // Будет работать с атрибутами объекта p1
```

### 2) Вызов метода из другой функции-члена того же класса

В этом случае вызов осуществляется напрямую, без использования точечной нотации (компилятор уже знает, какой объект обрабатывается).

Пример такого вызова увидим далее (глава конструкторов и деструкторов).

## Конструкторы и деструкторы

В предыдущем примере атрибуты объекта `p1` устанавливались вручную, однако опытный программист никогда бы не стал использовать такой подход. Помимо того, что в этом случае код подвержен ошибкам (можно забыть указать значение одного из атрибутов), существует гораздо более удобный способ это сделать, а именно использование конструктора.

**Конструктор** инициализирует объект при его создании. Он имеет такое же имя, что и сам класс, и синтаксически подобен функции. Однако в определении конструкторов не указывается тип возвращаемого значения. Синтаксис конструктора следующий:

```cpp
имя_класса() {
    // код конструктора
}
```

Обычно конструктор применяется, чтобы придать переменным экземпляра, определённым в классе, начальные значения или выполнить исходные действия, которые требуются для создания полностью сформированного объекта.

Дополнением к конструктору является **деструктор**. Во многих случаях при уничтожении объекту нужно выполнить некоторые действия или даже последовательность действий. Добиться этого поможет деструктор.

*Локальные переменные удаляются при выходе из блока, глобальные — после завершения программы.*

Имя деструктора совпадает с именем конструктора, но предваряется символом `~`. Подобно конструкторам, деструкторы не возвращают никаких значений.

Добавим конструктор и деструктор в наш код. Обратите внимание на реализацию кода внутри конструктора. Мы используем вызов метода класса внутри другой функции-члена.

```cpp
class Point {
public:
    float x;
    float y;
    float z;

    Point();    // Вызывает метод reset
    void reset();
    ~Point();   // Выводит сообщение об удалении
};

Point::Point() {
    reset();
    cout << "x = 0 ; y = 0 ; z = 0 \n";
}

void Point::reset() {
    x = 0; y = 0; z = 0;
}

Point::~Point() {
    cout << "Destructing...\n";
}
```

## Параметризированные конструкторы

В предыдущем примере использовался конструктор без параметров. Но, как правило, приходится иметь дело с конструкторами, которые принимают один или несколько параметров. Параметры в конструктор вносятся точно так же, как и в функцию: для этого достаточно объявить их внутри круглых скобок после его имени. Например, если мы хотим задать начальные координаты точки не равные (0 ; 0 ; 0), то параметризированный конструктор поможет нам это сделать:

```cpp
Point::Point(float X, float Y, float Z) {
    x = X; // Обязаны инициализировать атрибуты сразу при объявлении
    y = Y;
    z = Z;
}
```

Не забываем изменить прототип функции-члена: `Point(float X, float Y, float Z);`

Теперь создание экземпляра класса будет запрашивать обязательные аргументы:

```cpp
Point p1(2.0, 4.0, 6.0); // Устанавливаем точку в координаты (2 ; 4 ; 6)
```

Есть и вторая форма записи, но она используется реже из-за своего размера:

```cpp
Point p1 = Point(2.0, 4.0, 6.0);
```

*Формально между двумя этими записями есть небольшое различие, которое будет описано позже, но оно не влияет на результат выполнения нашей программы и многих других программ.*

Существует ещё одна форма инициализации объекта. Она применима только тогда, когда мы передаём единственный параметр в конструктор:

```cpp
Название_класса Имя_переменной = значение // Запись ob = x идентична записи ob(x)
```

## Встраиваемые функции

Прежде чем продолжить говорить о классах, сделаем важное отступление и поговорим о важном средстве C++, которое довольно часто используется в определении классов. Это **встраиваемая** или **подставляемая функция**. Встраиваемой называется функция, код которой подставляется в то место строки программы, из которого она вызывается, то есть вызов такой функции заменяется её кодом. Существует два способа создания функции таким способом.

1) Использование модификатора `inline`:

```cpp
inline int f() {
    // ...
}
```

Причина существования встраиваемых функций — эффективность. При каждом вызове обычной функции должна быть выполнена целая последовательность инструкций, связанных с обработкой самого вызова (в том числе отправка её аргументов в стек) и с возвратом из функции. Иногда компилятор тратит больше времени на эти аспекты, чем на произведение самих расчётов внутри функций. Однако если размер встраиваемой функции будет слишком большой, то размер программы может существенно увеличиться. При всех плюсах этого подхода компилятор не обязан встраивать функцию в код, а иногда и наоборот может встроить её без данного модификатора. Ниже описано моё мнение по использованию модификатора `inline`:

> "Явное использование `inline` исключительно для оптимизации небольших функций в современном C++ считается устаревшим хорошим тоном. Мы полагаемся на оптимизатор компилятора. Однако `inline` остаётся крайне полезным инструментом для решения проблемы множественного определения (ODR), когда мы вынуждены помещать реализацию функции в заголовочный файл. Поэтому я использую `inline` как техническое средство для линковки, а не как директиву ускорения."

### Создание встраиваемых функций внутри класса

Любая функция, которая определяется внутри объявления класса, автоматически становится встраиваемой и будет обладать свойствами, описанными ранее.

Пример автоматически встраиваемой функции:

```cpp
class Hello {
public:
    void f() { cout << "Hello world"; } // Автоматически встраиваемая функция
};
```

Небольшие (как в этом примере) функции обычно определяются в объявлении класса. Применение «внутриклассовых» встраиваемых публичных функций — обычная практика в программировании на C++, поскольку с их помощью обеспечивается доступ к приватным переменным. Такие функции называются **получающими**, или **функциями доступа**. Успех ООП зачастую определяется умением управлять доступом к данным посредством функций-членов. Поскольку большинство программистов на C++ определяют функции доступа внутри своих классов, это соглашение применяется и к остальным примерам данной книги. То же самое я рекомендую делать и вам.

Дополним наш класс `Point`, используя функции доступа:

```cpp
#include <iostream>

using namespace std;

class Point {
    float x;
    float y;
    float z;
public:
    Point(float X, float Y, float Z); // Вызывает метод reset

    float get_x() { return x; } // Не забывайте, все функции, объявленные в классе имеют доступ к приватным переменным
    float get_y() { return y; } // Понятие 'функция доступа' - лишь способ получения значений приватных переменных через функцию.
    float get_z() { return z; } // Поэтому функции доступа не обязаны быть встраиваемыми

    void reset();
    ~Point(); // Выводит сообщение об удалении
};

int main() {
    Point p1 = Point(2.0, 4.0, 6.0);
    cout << p1.get_x() << ' ' << p1.get_y() << ' ' << p1.get_z() << '\n';
}

Point::Point(float X, float Y, float Z) {
    x = X;
    y = Y;
    z = Z;
}

inline void Point::reset() {
    x = 0; y = 0; z = 0;
}

Point::~Point() {
    cout << "Destructing...\n";
}
```

## Массивы объектов

Массивы объектов можно создавать аналогично массивам значений других типов. Ниже приведён пример создания массива объектов типа `Point`, доступ к элементам этого массива осуществляется через индексирование массива.

```cpp
class Point {
public:
    float x;
    float y;
    float z;
};

int main() {
    Point points[3];
    int i;
    for (i = 0; i != 3; ++i) {
        points[i].x = i;
        cout << points[i].x << ' ';
    }
}
```

Если класс включает параметризированный конструктор, то массив объектов этого класса можно инициализировать.

### Варианты инициализации

1) Если передаём единственное значение для всех точек:

```cpp
Point points[3] = {1, 2, 3};
```

В действительности синтаксис инициализации массива, выраженный строкой `Point points[3] = {1, 2, 3}`, представляет собой сокращённый вариант такой записи:

```cpp
Point points[3] = { Point(1), Point(2), Point(3) };
```

2) Если передаём более одного значения или мы используем многомерный массив, то используется только вторая форма записи.

Пример с запросом трёх аргументов для атрибутов:

```cpp
#include <iostream>

using namespace std;

class Point {
public:
    Point(float X, float Y, float Z);

    float x;
    float y;
    float z;
};

int main() {
    Point points[3] = { Point(1, 2, 3), Point(3, 2, 1), Point(0, 0, 0) };
    for (int i = 0 ; i != 3; ++i) {
        cout << points[i].x << ' ' << points[i].y << ' ' << points[i].z << '\n';
    }
}

Point::Point(float X, float Y, float Z) {
    x = X;
    y = Y;
    z = Z;
}
```

## Указатели на объекты

Доступ к объекту можно получить напрямую или через указатель на него. Чтобы получить доступ к отдельному элементу (члену) объекта через указатель, следует воспользоваться операцией «стрелка» (`->`).

Синтаксис объявления указателя на объект аналогичен объявлению указателей на другие типы данных. В приведённом ниже примере создаётся класс `P_example`, объявляется объект `ob` этого класса и указатель `p` на объект типа `P_example`. Фрагмент демонстрирует два способа доступа к объекту: прямой — к самому экземпляру `ob`, и косвенный — через указатель.

```cpp
// Простой пример использования указателя на объект

#include <iostream>
using namespace std;

class P_example {
    int num;
public:
    void set_num(int val) { num = val; }
    void show_num() { cout << num << "\n"; }
};

int main()
{
    P_example ob, *p;      // Объявляем объект и указатель на него
    ob.set_num(1);         // Прямой доступ к объекту ob
    ob.show_num();

    p = &ob;               // Присваиваем указателю p адрес объекта ob
    p->set_num(20);        // Вызов метода через указатель (операция «стрелка»)
    p->show_num();

    return 0;
}
```

Адрес объекта `ob` получают с помощью операции взятия адреса `&` — этот механизм работает так же, как и для переменных любых других типов.

Известно, что при инкрементировании или декрементировании указателя его значение корректируется так, чтобы он указывал на следующий или предыдущий элемент базового типа. Тот же принцип действует и для указателей на объекты: после увеличения или уменьшения адреса указатель будет ссылаться на соседний объект соответствующего типа. Чтобы наглядно показать работу этого механизма, можно модифицировать приведённую программу — например, вместо одного объекта `ob` объявить массив объектов и затем перемещать указатель по элементам массива.

```cpp
// Инкрементация и декрементация указателя на объект

#include <iostream>
using namespace std;

class P_example {
    int num;
public:
    void set_num(int val) { num = val; }
    void show_num() { cout << num << "\n"; }
};

int main()
{
    P_example ob[2], *p;

    ob[0].set_num(10);   // Прямой доступ к первому элементу
    ob[1].set_num(20);   // Прямой доступ ко второму элементу

    p = &ob[0];          // Указатель p получает адрес первого объекта
    p->show_num();       // Выведет 10

    p++;                 // Сдвигаем указатель на следующий объект (ob[1])
    p->show_num();       // Выведет 20

    p--;                 // Возвращаемся к предыдущему объекту (ob[0])
    p->show_num();       // Снова выведет 10

    return 0;
}
```

**Результат выполнения программы:**
```
10
20
10
```

**Ключевые моменты:**
- При инкременте (`p++`) указатель перемещается на **следующий объект** того же типа, то есть его адрес увеличивается на `sizeof(P_example)`.
- При декременте (`p--`) указатель смещается на **предыдущий** объект.
- Работает это точно так же, как с указателями на встроенные типы (например, `int*`).
- Такой приём удобен при обходе массивов объектов, особенно в циклах.

## Перегрузка конструкторов

Конструкторы класса можно перегружать, что даёт гибкость инициализации атрибутов класса. Сделать это можно так же, как с обычными функциями.

---

## Присваивание объектов

Если два объекта имеют одинаковый тип (класс), то один из них можно присвоить другому. Для присваивания недостаточно, чтобы два класса были физически подобны; должны совпадать их имена. Если один объект присваивается другому, то по умолчанию данные первого объекта побитово копируются в другой. Таким образом, после этого объекты будут идентичными, но отдельными. Изменения одного объекта не отразятся на изменении другого.

Однако есть опасный момент:

Например, если объект `a` имеет указатель на объект `b` и объект `a` имеет собственную копию, то изменение объекта `b` повлияет сразу на оба объекта. В таком случае, как будет показано позже, необходимо заблокировать создание побитовой копии путём определения собственной операции присваивания для данного класса.

---

## Передача объектов функциям

Объект можно передать функции путём передачи по значению и через ссылку. Рассмотрим первый вариант.

При передаче объекта функции создаётся его копия. Здесь возникает два вопроса:

1. Вызывается ли конструктор объекта при его копировании?
2. Вызывается ли деструктор объекта при его уничтожении?

Когда при вызове функции создаётся копия аргумента, обычный конструктор **не вызывается**. Вместо него вызывается **конструктор копирования** объекта, который определяет, как должна быть создана копия объекта. (Как создавать конструктор копирования будет описано позднее.) Но если в классе явно не определён конструктор копирования, C++ предоставляет его автоматически. Поскольку для инициализации некоторых аспектов объекта используется обычный конструктор, он не должен вызываться для создания копии уже существующего объекта, иначе такой вызов изменил бы его состояние (мы бы вернулись к начальным значениям объекта).

Когда функция завершается и уничтожается копия объекта, вызывается деструктор этого объекта. Такая необходимость связана с выходом объекта из области видимости.

Несмотря на то, что в случае, когда объекты передаются функциям «по значению», здесь всё-таки возможен побочный эффект, связанный с повреждением объекта, используемого в качестве аргумента. Например, если такой объект требует динамического выделения памяти и освобождает её, когда уничтожается, его локальная копия при вызове деструктора освободит ту же самую область памяти. Описанная ситуация обычно приводит к повреждению оригинального объекта. Эту проблему решает вызов по ссылке (использование указателя или ссылочных параметров).

---

## Передача объектов по ссылке

При передаче объекта по ссылке функция будет обрабатывать сам объект, а не его копию => вносимые в параметр изменения автоматически влияют на аргумент.

---

## Возвращение объектов функциями

Если функция возвращает объект, автоматически создаётся временный объект при помощи конструктора копирования, который хранит возвращаемое значение. Именно этот объект реально и возвращается функцией. После возвращения значения объект уничтожается, вызывая деструктор (при его наличии). Здесь снова натыкаемся на угрозу удаления важной памяти деструктором.

---

## Создание конструкторов копирования

Иногда поведение конструктора по умолчанию не удовлетворяет нашим запросам. Однако C++ позволяет нам создавать собственные конструкторы копирования. Для начала уточним разницу между двумя разными случаями передачи одного значения другому: присваивание и инициализация.
Инициализация может выполняться когда:
* Один объект явно инициализируется при помощи другого:
`Myclass x = y`
* Копия объекта передаётся параметру функции:
`func1(y)`
* Генерируется временный объект (чаще всего в виде значения, возвращаемого функцией):
`y = func2()`

Помните: конструктор копирования не вызывается, когда значение одного объекта присваивается другому. К примеру, при выполнении следующего кода конструктор копирования вызван не будет:
```cpp
Myclass x;
Myclass y;
x = y; // Объекты уже были инициализированы.
```

Типичный синтаксис конструктора копирования внутри класса выглядит так:
```cpp
имя_класса(const имя_класса &объект) {
    // Тело конструктора
}
```

## Дружественные функции

В общем случае только члены класса имеют доступ к приватным атрибутам класса. Однако в C++ есть механизм, позволяющий предоставлять такой доступ функциям, которые не являются членами этого класса. Для этого достаточно объявить их дружественными по отношению к этому классу.
Чтобы сделать функцию другом, включите её прототип в раздел `public` объявления класса и предварите его ключевым словом `friend`.
Пример:
```cpp
class Myclass {
    //...
public:
    friend void f(); // функция может быть другом нескольких классов сразу.
}
```

Доступ к членам класса внутри функции осуществляется при помощи точечной нотации. Поскольку такие функции не являются методами классов, то при вызове таких функций их не нужно квалифицировать с указанием имени объекта (и даже запрещено).

**Плюсы использования дружественных функций:**
* Дружественные функции могут быть полезны при перегрузке определённых операций.
* Они упрощают создание некоторых функций ввода/вывода.
* В некоторых случаях два (или более) класса могут содержать члены, находящиеся во взаимной связи с другими частями программы.

Пример последнего:

```cpp
class Cylinder; // опережающее объявление

enum colors { red, green, yellow };

class Cube {
    colors color;
public:
    Cube(colors c) { color = c; }
    friend bool sameColor(Cube x, Cylinder y);
};

class Cylinder {
    colors color;
public:
    Cylinder(colors c) { color = c; }
    friend bool sameColor(Cube x, Cylinder y);
};

bool sameColor(Cube x, Cylinder y)
{
    if (x.color == y.color)
        return true;
    else
        return false;
}
```

Заметьте, что в этой программе используется опережающее объявление для класса `Cylinder`. Его необходимость заключается в том, что объявление функции `sameColor()` в классе `Cube` принимает ссылку на класс `Cylinder` до его объявления.

Дружеский элемент одного класса может быть членом другого. Перепишем предыдущую программу так, чтобы функция `sameColor()` стала членом класса `Cube`. Обратите внимание на использование операции разрешения области видимости при объявлении функции `sameColor()` дружественной классу `Cylinder`.

```cpp
class Cylinder; // опережающее объявление

enum colors { red, green, yellow };

class Cube {
    colors color;
public:
    Cube(colors c) { color = c; }
    bool sameColor(Cylinder y);
};

class Cylinder {
    colors color;
public:
    Cylinder(colors c) { color = c; }
    friend bool Cube::sameColor(Cylinder y);
};

bool Cube::sameColor(Cylinder y)
{
    if (color == y.color)
        return true;
    else
        return false;
}
```

Функция `sameColor()` является членом класса `Cube`, поэтому она должна вызываться для объектов класса `Cube`, то есть она имеет прямой доступ ко всем членам этого класса.

## Структуры и объединения

В дополнение к ключевому слову `class` в C++ есть возможность создания двух классовых типов: структуры и объединения.

### Структуры

Структуры перешли в C++ из языка C. Объявление структуры синтаксически похоже на объявление класса. В C структура могла содержать только поля данных, но в C++ это ограничение снято: структура фактически является альтернативной формой класса. Единственное принципиальное отличие — уровень доступа по умолчанию: в `struct` все члены публичны, а в `class` — приватны. Во всём остальном они эквивалентны. Пример:
```cpp
struct Test {
    int a; // Публичны по умолчанию
    int b;
private:
    int c; // приватны
};
```

Возникает закономерный вопрос: зачем в языке существуют оба механизма, если они столь похожи? Это связано с историческим решением создать C++ как надстройку над C, сохранив полную совместимость. В C все поля структур всегда были общедоступными (понятий приватности и публичности там не было). Чтобы не нарушать обратную совместимость, в C++ для структур оставили публичный доступ по умолчанию. Новое ключевое слово `class` было введено специально для поддержки инкапсуляции, поэтому его члены по умолчанию приватны. Кроме того, разделение типов даёт пространство для будущей эволюции языка: классы могут развиваться синтаксически независимо от структур, не обременяя себя обязательствами совместимости с C.

На практике программисты на C++ часто придерживаются негласного правила: `class` используют для объектов с поведением (функциями-членами), а `struct` — для простых пассивных структур данных, иногда называемых POD (Plain Old Data — «простая старая память»).

### Объединения

Объединение — это специальный тип данных, в котором все члены данных занимают одну и ту же область памяти. В любой момент времени в объединении хранится только одно значение, но вы можете интерпретировать его по‑разному — как целое число, символы, дробное число и т.д.
Пример:
```cpp
union utype {
    short int i;   // 2 байта
    char ch;       // 1 байт
};
```

Здесь `i` и `ch` накладываются друг на друга в памяти. Если вы запишете число в `i`, то при чтении `ch` вы получите первый байт этого числа (и наоборот). Размер объединения равен размеру самого большого члена (в примере — 2 байта).

Пример с двумя символами:
```cpp
union u_type {
    short int i;    // 2 байта
    char ch[2];     // тоже 2 байта (массив из двух символов)
};
```

Если записать в `i` число 1000, то его байты можно прочитать как два символа `ch[0]` и `ch[1]`.
Если записать два символа `'x'` и `'y'` в `ch`, то при чтении `i` получится число, составленное из этих байтов (в примере вышло 22872).

*Объединение — это класс (class-type) с рядом жёстких ограничений (доступ к его членам осуществляется по таким же правилам, что и у обычных классов).*

**Что можно и что нельзя в C++**
Можно:
- Включать конструкторы и обычные функции-члены.
- Использовать объединение как полноценный класс (но с ограничениями).
- Управлять доступом через `public`/`private` (по умолчанию все члены объединения публичны).

Нельзя:
- Наследовать от объединения или делать его базовым классом.
- Иметь виртуальные методы.
- Использовать статические переменные, ссылки, объекты с перегруженным `operator=`, объекты с явным конструктором/деструктором.

**Зачем это нужно?**
- Экономия памяти — когда вы точно знаете, что в каждый момент используется только один вариант данных.
- Удобная интерпретация сырых байтов — например, работа с сетевыми пакетами, двоичными файлами, аппаратными регистрами.

### Анонимные объединения

В языке C++ предусмотрен специальный тип объединения, который называется **анонимным**. Анонимное объединение не имеет наименования типа, и поэтому его объект объявить невозможно. При этом оно сообщает компилятору о том, что его члены разделяют одну и ту же область памяти. Обращение к самим переменным объединения происходит непосредственно, без использования операции `.` (точка).
Рассмотрим такой пример:

```cpp
// Пример использования анонимного объединения
#include <iostream>
#include <cstring>
using namespace std;

int main()
{
    // Определяем анонимное объединение
    union {
        long l;
        double d;
        char s[4];
    };
    
    // К членам анонимного объединения выполняется обращение напрямую
    l = 100000;
    cout << l << " ";
    d = 123.2342;
    cout << d << " ";
    strcpy(s, "hi");
    cout << s;

    return 0;
}
```

Как вы можете убедиться, к элементам объединения можно получить доступ так же, как к обычным переменным. Хотя они объявлены как часть анонимного объединения, их имена находятся на том же уровне области видимости, что и другие локальные переменные того же блока. Поэтому, чтобы избежать конфликта имён, член анонимного объединения не должен иметь имя, совпадающее с именем любой другой переменной, которая объявлена в той же области видимости.

Все описанные выше ограничения, налагаемые на использование объединений, применимы и к анонимным объединениям. К ним нужно добавить ещё несколько. Анонимные объединения должны содержать только члены данных (функции-члены недопустимы). Также они не могут включать приватные или защищённые элементы (модификатор `protected` описан в модуле 10). Наконец, глобальные анонимные объединения необходимо объявлять с использованием модификатора `static`.

## Ключевое слово this

Прежде чем переходить к теме перегрузки операций, рассмотрим ключевое слово `this`. При каждом вызове функции-члена ей автоматически передаётся указатель, именуемый `this`, на объект, для которого она вызывается. Указатель `this` — это неявный параметр, принимаемый всеми функциями-членами. Таким образом, в любой функции-члене указатель `this` можно использовать для ссылки на вызывающий объект.

Как вам известно, функция-член может иметь прямой доступ к приватным (`private`) членам данных своего класса. Допустим, у нас определён такой класс:

```cpp
class Test {
    int i;
    void f() { ... };
    // ...
};
```

В функции `f()` для присваивания члену `i` значения 10 можно использовать следующий код:

```cpp
i = 10;
```

На деле предыдущая инструкция представляет собой сокращённую форму следующей:

```cpp
this->i = 10;
```

Чтобы понять, как работает указатель `this`, рассмотрим короткую программу:

```cpp
// Использование указателя this

#include <iostream>
using namespace std;

class Test {
    int i;
public:
    void load_i(int val) {
        this->i = val;    // То же самое, что i = val;
    }
    int get_i() {
        return this->i;    // То же самое, что return i;
    }
};

int main()
{
    Test o;

    o.load_i(100);
    cout << o.get_i();

    return 0;
}
```

При выполнении эта программа отображает число 100. Конечно, предыдущий пример тривиален, но в нём показано, как можно использовать указатель `this`.

И ещё один момент. Дружественные функции не имеют указателя `this`, поскольку они не являются членами класса. Только функции-члены имеют этот указатель.

## Перегрузка операций

**Перегрузка операций** позволяет задать поведение стандартных операторов (+, -, ++, ==, = и др.) для объектов пользовательских классов. Это делает новые типы данных полноценными участниками языка.

- **Операторная функция** — специальная функция вида `возвращаемый_тип operator@(параметры)`, где `@` — символ оператора.
- Может быть **членом класса** или **не-членом** (часто объявляется `friend`).
- Перегрузка **не меняет** приоритет операции, арность (количество операндов) и ассоциативность.

### Перегрузка через функции-члены

- У функций-членов есть неявный параметр `this`, указывающий на вызывающий объект.
- **Бинарная операция** (например, `+`):
  - Левый операнд → объект, вызвавший функцию (передаётся через `this`).
  - Правый операнд → единственный явный параметр.
- **Унарная операция** (например, `++`):
  - Не имеет явных параметров (префиксная форма).
  - Работает с `this`.

#### Пример для класса `ThreeD` (координаты x, y, z)

```cpp
class ThreeD {
    int x, y, z;
public:
    ThreeD(int i, int j, int k) : x(i), y(j), z(k) {}
    
    // Перегрузка + (бинарный)
    ThreeD operator+(ThreeD op2) {
        ThreeD temp;
        temp.x = x + op2.x;   // x = this->x
        temp.y = y + op2.y;
        temp.z = z + op2.z;
        return temp;          // новый объект, не изменяя операндов
    }
    
    // Перегрузка = (бинарный)
    ThreeD operator=(ThreeD op2) {
        x = op2.x; y = op2.y; z = op2.z;
        return *this;         // возвращаем изменённый объект
    }
};
```

**Важно:**
- `operator+` не должен изменять операнды (как и для встроенных типов).
- `operator=` изменяет левый операнд и возвращает `*this` для цепочек (`a = b = c`).
- Для составных выражений (`a + b + c`) оператор должен возвращать объект того же класса.

---

### Указатель `this`

- Неявный параметр всех **нестатических** функций-членов.
- Указывает на объект, для которого вызвана функция.
- Используется для доступа к членам: `this->x` или просто `x` (сокращённо).
- **Дружественные функции** `this` не имеют.

---

### Префиксная и постфиксная формы `++`/`--` (члены класса)

- **Префиксная** (`++obj`) — функция без параметров:
  ```cpp
  ThreeD operator++() {
      ++x; ++y; ++z;
      return *this;   // возвращаем изменённый объект
  }
  ```
- **Постфиксная** (`obj++`) — функция с фиктивным параметром `int`:
  ```cpp
  ThreeD operator++(int notused) {
      ThreeD temp = *this;   // сохраняем старое значение
      ++x; ++y; ++z;
      return temp;           // возвращаем старое значение
  }
  ```

Компилятор различает их по наличию/отсутствию фиктивного параметра.

---

### Перегрузка через дружественные функции (не-члены)

- Не имеют `this` → оба операнда передаются явно.
- Позволяют **левому операнду** быть встроенным типом (например, `int + obj`).
- **Нельзя** перегружать `=`, `()`, `[]`, `->` через не-члены (только члены класса).

#### Пример бинарного `+` для `ThreeD + int` и `int + ThreeD`

```cpp
class ThreeD {
    // ...
    friend ThreeD operator+(ThreeD op1, int op2);
    friend ThreeD operator+(int op1, ThreeD op2);
};

// obj + int
ThreeD operator+(ThreeD op1, int op2) {
    ThreeD temp;
    temp.x = op1.x + op2;
    temp.y = op1.y + op2;
    temp.z = op1.z + op2;
    return temp;
}

// int + obj
ThreeD operator+(int op1, ThreeD op2) {
    ThreeD temp;
    temp.x = op2.x + op1;
    temp.y = op2.y + op1;
    temp.z = op2.z + op1;
    return temp;
}
```

Теперь работают оба варианта: `a + 10` и `10 + a`.

---

### Перегрузка унарных операций с помощью дружественных функций

При перегрузке `++` или `--` через дружественную функцию **обязательно передавать операнд по ссылке**, чтобы изменения отразились на исходном объекте.

- **Префиксная форма** (один параметр):
  ```cpp
  ThreeD operator++(ThreeD &op1) {
      op1.x++; op1.y++; op1.z++;
      return op1;   // возврат изменённого объекта
  }
  ```
- **Постфиксная форма** (два параметра, второй – фиктивный `int`):
  ```cpp
  ThreeD operator++(ThreeD &op1, int notused) {
      ThreeD temp = op1;   // сохраняем старое значение
      op1.x++; op1.y++; op1.z++;
      return temp;         // возвращаем старое значение
  }
  ```

**Запомните:** без ссылки изменения не будут видны снаружи.

---

### Перегрузка операций отношения (`==`, `<` и др.)

- Обычно возвращают `bool` (или `int` для совместимости).
- Это позволяет использовать их в условных выражениях.

#### Пример перегрузки `==` для `ThreeD`

```cpp
bool ThreeD::operator==(ThreeD op2) {
    return (x == op2.x) && (y == op2.y) && (z == op2.z);
}
```

Теперь можно писать:
```cpp
if (a == b) cout << "Equal\n";
```

Аналогично перегружаются `<`, `>`, `!=` и другие.

# Перегрузка составных операторов присваивания (`+=`, `-=` и др.)

Во время выполнения составных операций присваивания происходят **2 вещи** (на примере операции `+=`):  
При выполнении `a += b` для пользовательского класса:

1) К текущему объекту (`*this`) прибавляются значения полей объекта `b` (правый операнд). Сам объект `*this` изменяется на месте.  
2) После изменения возвращается ссылка на этот же объект (`*this`).

**Пример:**  
```cpp
int a = 1;
cout << (a += 1); // Спокойно выведет 2, т.к ссылка — всего лишь псевдоним переменной, она обладает всеми свойствами своей переменной.
```

Следовательно, для перегрузки таких операций мы должны сделать эти два пункта.  
Пример для класса `Point`:

```cpp
Point& operator+=(const Point& p) { 
    x += p.x; 
    y += p.y;
    z += p.z;
    return *this; // Разыменовываем указатель, позже он преобразуется в ссылку (избегаем создания копии в возврате)
}
```

Подобный синтаксис может вызвать множество вопросов:

1) **Почему мы должны придерживаться идиом C++ и возвращать ссылку?**  
   Существует несколько причин, одна из них: поскольку операции `+=` можно комбинировать (`a += b += c...`), то после каждого вызова оператора мы должны сохранить структуру вызова для второго: слева — ссылка (`this` — инициатор вызова), справа — второй операнд. Именно возвращая ссылку, мы оставляем себе возможность комбинировать данную операцию.

2) **Почему именно `const Point &p`?**  
   Во-первых, использование ссылочного параметра помогает избежать копирования данных (ускорение работы программы, уменьшение затрат ресурсов). Во-вторых, модификатор `const` защищает нас от случайного изменения объекта `p`.

### Важные ограничения и советы

**Ограничения языка:**
- Нельзя перегружать операторы: `.`, `.*`, `::`, `?:`, `sizeof` (и некоторые другие, полный список см. в стандарте).
- **Нельзя** менять приоритет, ассоциативность или количество операндов.
- **Операторные функции не могут иметь аргументы по умолчанию** (кроме `operator()` – но это особый случай).
- Некоторые операторы (`=`, `()`, `[]`, `->`) **обязаны** быть членами класса.

**Советы по стилю:**
- Старайтесь сохранять **естественный смысл** оператора (не делайте из `+` вычитание).
- Для симметричных операций (например, `+`) используйте дружественные функции, чтобы поддерживать `int + obj`.
- Для модифицирующих операций (`=`, `+=`, `++`) предпочтительнее функции-члены.
- Всегда возвращайте объекты по значению или ссылке так, чтобы поддерживались цепочки выражений.
