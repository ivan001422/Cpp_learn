# 📘 Отображения (ассоциативные массивы) — `std::map` и `std::unordered_map`

Отображение (map) — это контейнер, хранящий пары **ключ–значение**. Ключи уникальны, каждому ключу сопоставлено некоторое значение. Отображение можно представить как обобщение массива: если в обычном массиве индекс — это целое число, то здесь ключом может быть любой тип (строки, числа, пользовательские типы). Доступ к значению происходит по ключу, аналогично массиву через `operator[]`.

C++ предлагает два основных варианта:
- `std::map` — упорядоченное отображение на основе красно-чёрного дерева (операции O(log n)).
- `std::unordered_map` — неупорядоченное отображение на основе хеш-таблицы (среднее время O(1)).

---

## Подключение и объявление

```cpp
#include <map>      // для std::map
#include <unordered_map> // для std::unordered_map
```

Объявление map:

```cpp
std::map<std::string, int> m; // ключи — строки, значения — целые
std::map<int, double, std::greater<int>> m2; // ключи по убыванию
```

Для unordered_map:

```cpp
std::unordered_map<std::string, int> um; // хеш-таблица
```

Инициализация списком:

```cpp
std::map<std::string, int> m = {
    {"apple", 5},
    {"banana", 3},
    {"cherry", 7}
};
```

## Основные методы

| Метод                       | Описание                                                                 | Сложность (map) | Сложность (unordered_map) |
|-----------------------------|--------------------------------------------------------------------------|-----------------|---------------------------|
| `operator[](key)`           | Возвращает ссылку на значение по ключу. Если ключа нет, создаёт пару со значением по умолчанию. | O(log n)        | O(1) среднее |
| `at(key)`                   | Возвращает ссылку на значение, бросает исключение при отсутствии ключа.    | O(log n)        | O(1) среднее |
| `insert({key, value})`      | Вставляет пару. Если ключ уже есть, ничего не делает (для map).           | O(log n)        | O(1) среднее |
| `erase(key)`                | Удаляет элемент по ключу. Возвращает количество удалённых (0 или 1).       | O(log n)        | O(1) среднее |
| `erase(iterator)`           | Удаляет элемент по итератору.                                             | O(1) амортиз.   | O(1) среднее |
| `find(key)`                 | Возвращает итератор на пару или `end()`, если ключ не найден.             | O(log n)        | O(1) среднее |
| `count(key)`                | Возвращает 1, если ключ есть, иначе 0 (для map).                           | O(log n)        | O(1) среднее |
| `size()`, `empty()`, `clear()` | Аналогично другим контейнерам.                                        | O(1), O(n)      | O(1), O(n) |
| `begin()`, `end()`          | Итераторы для обхода. В `map` элементы отсортированы по ключу.             | O(1)            | O(1) |

В `unordered_map` итераторы не гарантируют порядка, но позволяют обход всех элементов.

## Использование `operator[]` — важные нюансы

`operator[]` автоматически создаёт элемент, если его нет. Значение инициализируется значением по умолчанию (0 для чисел, пустая строка и т.п.). Это удобно, но может привести к неожиданным вставкам при проверке наличия ключа.

**Пример:**
```cpp
std::map<std::string, int> m;
if (m["apple"] == 0) { // если apple нет, будет создан с 0
    std::cout << "Apple not found, but now it's inserted with 0\n";
}
```

Поэтому для проверки существования лучше использовать `find()`:

```cpp
auto it = m.find("apple");
if (it != m.end()) {
    std::cout << "Значение: " << it->second << '\n';
} else {
    std::cout << "Ключ не найден\n";
}
```

Или `count()` (но он не даёт доступа к значению).

## Пример использования `std::map`

```cpp
#include <iostream>
#include <map>

int main() {
    std::map<std::string, int> m;

    m["monkey"] = 4;
    m["banana"] = 3;
    m["harpsichord"] = 9;

    // Доступ
    std::cout << "banana: " << m["banana"] << '\n'; // 3

    // Проверка наличия
    if (m.count("apple") == 0)
        std::cout << "apple не найдено\n";

    // Вставка пары
    m.insert({"apple", 5}); // если apple нет, добавится

    // Обход всех элементов (в порядке ключа)
    std::cout << "Все элементы map:\n";
    for (const auto& pair : m) {
        std::cout << pair.first << " -> " << pair.second << '\n';
    }

    // Удаление
    m.erase("banana");

    return 0;
}
```

## `std::unordered_map` — хеш-таблица

Используется аналогично, но с некоторыми отличиями:

- Неупорядочен, итераторы проходят в недетерминированном порядке.
- Среднее время операций O(1), но в худшем случае (коллизии) может быть O(n).
- Для пользовательских типов в качестве ключей нужно определить хеш-функцию (по умолчанию есть для стандартных типов).
- Не поддерживает `lower_bound`/`upper_bound` (т.к. не упорядочен).

**Пример:**
```cpp
#include <iostream>
#include <unordered_map>

int main() {
    std::unordered_map<std::string, int> um;
    um["apple"] = 1;
    um["banana"] = 2;
    um["cherry"] = 3;

    for (const auto& p : um) {
        std::cout << p.first << " -> " << p.second << '\n'; // порядок произвольный
    }
    return 0;
}
```

---

## Обход и доступ к парам

Элементы хранятся как пары `std::pair<const Key, T>`. Доступ к ключу: `it->first`, к значению: `it->second`.

В циклах удобно использовать структурированные привязки (C++17):

```cpp
for (const auto& [key, value] : m) {
    std::cout << key << ": " << value << '\n';
}
```

## Когда использовать map vs unordered_map?

| Критерий                     | `std::map`                         | `std::unordered_map`           |
|------------------------------|------------------------------------|--------------------------------|
| Упорядоченность              | Да (по ключу)                      | Нет                            |
| Сложность операций           | O(log n)                           | O(1) в среднем, O(n) в худшем |
| Дополнительные методы        | `lower_bound`, `upper_bound`       | Отсутствуют                    |
| Память                       | Меньше накладных расходов          | Больше (хеш-таблица)           |
| Использование                | Если важен порядок ключей или диапазонные запросы | Если нужна максимальная скорость и порядок не важен |
| Работа с пользовательскими типами | Требуется оператор `<`          | Требуется хеш-функция и `==`  |

---

## Важные замечания

- В `std::map` и `std::unordered_map` ключи всегда уникальны. Если нужны несколько значений на один ключ, используйте `std::multimap` или `std::unordered_multimap`.
- При использовании `operator[]` с несуществующим ключом происходит вставка, поэтому не используйте его в константных контекстах или для проверки наличия.
- Итераторы на элементы не инвалидируются при вставке в `std::map` (кроме `end()`), а в `unordered_map` при рехешировании все итераторы инвалидируются.
- Для использования пользовательского типа в качестве ключа в `unordered_map` необходимо предоставить специализацию `std::hash<Key>` и оператор равенства.
