## Что такое итератор?

**Итератор** — это объект, который указывает на элемент в контейнере (например, в векторе, списке, массиве). Он ведёт себя похоже на указатель: его можно разыменовывать (`*it`) для доступа к значению и перемещать (`++it`) для перехода к следующему элементу.

Каждый контейнер предоставляет свои итераторы через методы:
- `begin()` — итератор на **первый** элемент.
- `end()` — итератор на **позицию за последним** элементом (не разыменовывается!).

```cpp
vector<int> v = {5, 2, 3, 1, 2, 5, 7, 1};
auto it = v.begin();   // указывает на 5
cout << *it << '\n';   // печатает 5
++it;                  // теперь указывает на 2
```

> **Важно:** `end()` указывает **за пределы** контейнера, поэтому использовать `*end()` нельзя. Обычно его используют как маркер окончания в циклах и алгоритмах.

## Итераторы и диапазоны

**Диапазон** — это непрерывная последовательность элементов, задаваемая двумя итераторами: первый указывает на начало, второй — на позицию **после** последнего элемента (полуоткрытый интервал `[first, last)`).

Например, итераторы `begin()` и `end()` определяют диапазон, охватывающий все элементы контейнера. Такой способ задания диапазона используется во всех стандартных алгоритмах.

```cpp
sort(v.begin(), v.end());         // сортируем весь вектор
reverse(v.begin(), v.end());      // меняем порядок на противоположный
random_shuffle(v.begin(), v.end()); // перемешиваем (устарело; используйте shuffle)
```

## Основные операции с итераторами

| Операция | Пример | Что делает |
|----------|--------|------------|
| **Разыменование** | `*it` | Возвращает ссылку на элемент, на который указывает итератор |
| **Инкремент** | `++it` или `it++` | Переход к следующему элементу |
| **Декремент** | `--it` или `it--` | Переход к предыдущему элементу (для двунаправленных итераторов) |
| **Арифметика** | `it + n`, `it - n` | Смещение на `n` позиций (для итераторов произвольного доступа) |
| **Сравнение** | `it1 == it2`, `it1 != it2` | Проверка, указывают ли итераторы на одну позицию |
| **Разность** | `it1 - it2` | Расстояние между итераторами (для итераторов произвольного доступа) |

Пример обхода через итераторы:

```cpp
for (auto it = v.begin(); it != v.end(); ++it) {
    cout << *it << " ";
}
```

## Разновидности итераторов

C++ предоставляет несколько категорий итераторов, каждая из которых поддерживает определённый набор операций:

| Категория | Поддерживаемые операции | Примеры контейнеров |
|-----------|--------------------------|----------------------|
| **Входной** (`InputIterator`) | Чтение, инкремент (одноразовый проход) | Потоки ввода |
| **Выходной** (`OutputIterator`) | Запись, инкремент | Потоки вывода, итераторы вставки |
| **Прямой** (`ForwardIterator`) | Чтение/запись, многократный проход | `forward_list`, `unordered_set` |
| **Двунаправленный** (`BidirectionalIterator`) | Прямой + декремент (`--it`) | `list`, `set`, `map` |
| **Произвольного доступа** (`RandomAccessIterator`) | Двунаправленный + арифметика (`it+n`, `it-n`, `it1-it2`) | `vector`, `deque`, `array` |

Для вектора доступны **итераторы произвольного доступа**, поэтому можно писать `v.begin() + 5` или сравнивать `it1 < it2`.

## Другие виды итераторов

- **Константные итераторы** (не позволяют изменять элементы):  
  `v.cbegin()`, `v.cend()` — возвращают `const_iterator`.
- **Обратные итераторы** (проход в обратном порядке):  
  `v.rbegin()` — указывает на последний элемент, `v.rend()` — на позицию перед первым.
- **Итераторы вставки** (для алгоритмов, вставляющих элементы):  
  `std::back_inserter(v)` — вставляет в конец, `std::front_inserter`, `std::inserter`.

Пример использования обратных итераторов:

```cpp
for (auto it = v.rbegin(); it != v.rend(); ++it) {
    cout << *it << " "; // выводит элементы в обратном порядке
}
```

## Стандартные алгоритмы и итераторы

Множество алгоритмов в `<algorithm>` работают с диапазонами. Вот некоторые полезные:

### Сортировка и перестановки
- `sort(first, last)` — сортировка по возрастанию.
- `reverse(first, last)` — разворот.
- `random_shuffle(first, last)` / `shuffle(first, last, gen)` — перемешивание.

### Поиск и бинарный поиск (требуют сортировки)
- `lower_bound(first, last, value)` — возвращает итератор на **первый** элемент, **не меньший** `value`.
- `upper_bound(first, last, value)` — возвращает итератор на **первый** элемент, **больший** `value`.

```cpp
vector<int> v = {2, 3, 3, 5, 7, 8, 8, 8};
sort(v.begin(), v.end());

auto a = lower_bound(v.begin(), v.end(), 5); // указывает на 5
auto b = upper_bound(v.begin(), v.end(), 5); // указывает на 7

cout << *a << " " << *b << '\n'; // 5 7
```

### Удаление дубликатов
Комбинация `sort`, `unique` и `erase` убирает повторяющиеся элементы:

```cpp
sort(v.begin(), v.end());
auto last = unique(v.begin(), v.end()); // перемещает уникальные в начало, возвращает итератор на новый конец
v.erase(last, v.end());                 // удаляем "хвост" с дубликатами
```

Теперь вектор содержит только уникальные элементы в отсортированном порядке.

## Инвалидация итераторов

Важно помнить, что итераторы могут стать **недействительными** после некоторых операций с контейнером:

- **Вставка** (`insert`, `push_back`) может вызвать реаллокацию памяти, что инвалидирует **все** итераторы (если `capacity` была исчерпана). Если реаллокации нет, инвалидируются только итераторы после позиции вставки.
- **Удаление** (`erase`, `pop_back`) инвалидирует итераторы на удалённые элементы и на элементы после них.
- **Очистка** (`clear`) инвалидирует все итераторы.

Практический совет: после модификации контейнера пересчитывайте итераторы (например, обновляйте их) или используйте алгоритмы, возвращающие новые итераторы.

## Преимущества использования итераторов

- **Универсальность** — одни и те же алгоритмы работают с разными контейнерами.
- **Эффективность** — итераторы оптимизированы для конкретного контейнера.
- **Безопасность** — границы проверяются через итераторы (например, `end()`).
- **Гибкость** — можно комбинировать алгоритмы и создавать сложные цепочки операций.
