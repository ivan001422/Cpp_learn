# 📘 Дек (двусторонняя очередь) — `std::deque`

Дек (от англ. *deque* — double-ended queue) — это структура данных, позволяющая эффективно добавлять и удалять элементы как в начале, так и в конце. В C++ она реализована в заголовочном файле `<deque>` и предоставляет удобный интерфейс для работы с двусторонней очередью.

---

## Что такое дек?

Дек — это динамический массив, который расширяется в обе стороны. В отличие от обычного вектора (`std::vector`), который эффективно работает только с концом, дек одинаково хорошо поддерживает операции с обоих концов. При этом доступ к любому элементу по индексу остаётся константным (как у массива).

Основные характеристики:
- **Вставка и удаление** в начале и в конце — за **O(1)** (амортизированно).
- **Произвольный доступ** по индексу — за **O(1)**.
- **Непрерывность памяти** не гарантируется, но доступ по индексу всё равно быстрый (обычно реализован как массив блоков).
- Память выделяется блоками, что позволяет избежать массового копирования при расширении.

---

## Подключение и объявление

Для работы с деком необходимо подключить заголовочный файл:

```cpp
#include <deque>
```

Объявление дека:

```cpp
std::deque<int> d;          // пустой дек целых чисел
std::deque<double> d2(5);   // 5 элементов, инициализированных нулями (для double)
std::deque<char> d3(3, 'a');// 3 элемента со значением 'a'
std::deque<int> d4 = {1, 2, 3, 4, 5}; // инициализация списком
```

## Основные методы

### Добавление элементов
| Метод | Описание | Пример |
|-------|----------|--------|
| `push_back(value)` | Добавляет элемент в конец | `d.push_back(10);` |
| `push_front(value)` | Добавляет элемент в начало | `d.push_front(5);` |
| `insert(pos, value)` | Вставляет элемент перед указанным итератором (O(n)) | `d.insert(d.begin()+2, 42);` |

### Удаление элементов
| Метод | Описание | Пример |
|-------|----------|--------|
| `pop_back()` | Удаляет последний элемент | `d.pop_back();` |
| `pop_front()` | Удаляет первый элемент | `d.pop_front();` |
| `erase(pos)` / `erase(first,last)` | Удаляет элемент или диапазон (O(n)) | `d.erase(d.begin()+1);` |
| `clear()` | Удаляет все элементы | `d.clear();` |

### Доступ к элементам
| Метод | Описание | Пример |
|-------|----------|--------|
| `front()` | Ссылка на первый элемент | `int first = d.front();` |
| `back()` | Ссылка на последний элемент | `int last = d.back();` |
| `operator[](index)` | Доступ по индексу (без проверки границ) | `d[2] = 7;` |
| `at(index)` | Доступ с проверкой границ (бросает исключение) | `d.at(2) = 7;` |

### Размер и состояние
| Метод | Описание | Пример |
|-------|----------|--------|
| `size()` | Количество элементов | `size_t n = d.size();` |
| `empty()` | Проверка на пустоту | `if (d.empty()) ...` |
| `resize(count)` | Изменяет размер, добавляя нули или удаляя | `d.resize(10);` |

### Итераторы
Дек предоставляет итераторы произвольного доступа, поэтому можно использовать все стандартные алгоритмы:

```cpp
for (auto it = d.begin(); it != d.end(); ++it)
    std::cout << *it << " ";
```

Также доступны `cbegin()`, `cend()` (константные), `rbegin()`, `rend()` (обратные).

---

## Пример использования

```cpp
#include <iostream>
#include <deque>

int main() {
    std::deque<int> d;

    // Добавление с обоих концов
    d.push_back(2);      // [2]
    d.push_back(5);      // [2, 5]
    d.push_front(3);     // [3, 2, 5]
    d.push_front(1);     // [1, 3, 2, 5]

    // Доступ к элементам
    std::cout << "Первый: " << d.front() << '\n';  // 1
    std::cout << "Последний: " << d.back() << '\n'; // 5
    std::cout << "d[2] = " << d[2] << '\n';         // 2

    // Удаление с обоих концов
    d.pop_front();       // [3, 2, 5]
    d.pop_back();        // [3, 2]

    // Вывод всех элементов через итераторы
    std::cout << "Дек: ";
    for (int x : d)
        std::cout << x << " "; // 3 2
    std::cout << '\n';

    return 0;
}
```

## Сложность операций

| Операция                | Сложность |
|-------------------------|-----------|
| `push_back`, `push_front`, `pop_back`, `pop_front` | **O(1)** (амортизированно) |
| Доступ по индексу (`[]`, `at`) | **O(1)** |
| Вставка/удаление в середине (`insert`, `erase`) | **O(n)** (сдвиг элементов) |
| `size`, `empty`, `front`, `back` | **O(1)** |

---

##  Когда использовать дек?

Дек идеально подходит в ситуациях, когда вам нужны операции вставки/удаления с обоих концов, а также произвольный доступ по индексу. Конкретные сценарии:

- **Очередь с двухсторонним доступом** (например, для реализации алгоритмов, требующих обработки с обоих концов).
- **Окна скользящего среднего** (поддержка добавления справа и удаления слева).
- **Хранение истории действий** (undo/redo), где нужна вставка и удаление с обоих концов.

Если вам нужны только операции с концом — предпочтительнее `std::vector` (меньше накладных расходов). Если вставка/удаление в середине — лучше `std::list`. Дек занимает промежуточное положение.

---

## Отличия от вектора и списка

| Характеристика          | `deque`              | `vector`             | `list`               |
|-------------------------|----------------------|----------------------|----------------------|
| Вставка/удаление в начале | **O(1)**            | O(n)                 | **O(1)**             |
| Вставка/удаление в конце | **O(1)**            | **O(1)** (аморт.)    | **O(1)**             |
| Доступ по индексу       | **O(1)**            | **O(1)**             | O(n)                 |
| Память                  | Блочная, не непрерывна | Непрерывный массив   | Фрагментированная    |
| Итераторы               | Произвольного доступа | Произвольного доступа | Двунаправленные      |

---

## Важные замечания

- Итераторы, ссылки и указатели на элементы дека **инвалидируются** при вставке/удалении в середине, но при вставке с обоих концов итераторы остаются действительными (за исключением `end()`).
- Дек не гарантирует, что все элементы хранятся в непрерывной памяти, поэтому некоторые операции (например, передача в функции, ожидающие массив) могут потребовать копирования.
- В отличие от `vector`, дек не предоставляет метод `reserve()` для предварительного выделения памяти, так как его внутренняя структура не требует непрерывного блока.
