# 📘 Очередь с приоритетом (`std::priority_queue`)

Очередь с приоритетом — это структура данных, которая позволяет эффективно:
- вставлять элементы (`push`),
- получать наибольший (или наименьший) элемент без удаления (`top`),
- удалять этот наибольший (или наименьший) элемент (`pop`).

В отличие от обычной очереди (FIFO), элементы извлекаются не в порядке поступления, а в порядке убывания (или возрастания) приоритета. Это реализовано на основе **кучи (heap)** — специального двоичного дерева, где операции вставки и удаления выполняются за логарифмическое время, а получение максимума/минимума — за константное.

В C++ очередь с приоритетом реализована как **адаптер контейнера** (по умолчанию на основе `std::vector`). Она определена в заголовочном файле `<queue>`.

---

## Подключение и объявление

```cpp
#include <queue>
```

### Обычная очередь (максимальный элемент на вершине)

```cpp
std::priority_queue<int> pq; // по умолчанию - max-heap
```

### Очередь для извлечения минимального элемента

```cpp
std::priority_queue<int, std::vector<int>, std::greater<int>> pq_min;
```

Здесь:
- `int` — тип элементов,
- `std::vector<int>` — базовый контейнер (можно также `deque<int>`),
- `std::greater<int>` — компаратор, который делает очередь минимальной (вершина — наименьший элемент).

Можно указать свой компаратор для пользовательских типов.

---

## Основные методы

| Метод | Описание | Сложность |
|-------|----------|-----------|
| `push(value)` | Вставляет элемент в очередь | O(log n) |
| `pop()` | Удаляет вершину (максимальный или минимальный элемент) | O(log n) |
| `top()` | Возвращает константную ссылку на вершину (без удаления) | O(1) |
| `empty()` | Проверяет, пуста ли очередь | O(1) |
| `size()` | Количество элементов | O(1) |

> **Важно:** В очереди с приоритетом нет итераторов, нет произвольного доступа, нельзя обойти все элементы без последовательного извлечения.

---

## Пример использования (max-heap)

```cpp
#include <iostream>
#include <queue>

int main() {
    std::priority_queue<int> pq;
    pq.push(3);
    pq.push(5);
    pq.push(7);
    pq.push(2);

    std::cout << "Вершина: " << pq.top() << '\n'; // 7
    pq.pop(); // удаляем 7
    std::cout << "Новая вершина: " << pq.top() << '\n'; // 5
    pq.pop(); // удаляем 5
    pq.push(6);
    std::cout << "Вершина: " << pq.top() << '\n'; // 6

    while (!pq.empty()) {
        std::cout << pq.top() << ' ';
        pq.pop();
    } // выведет "6 3 2" (в порядке убывания)
    return 0;
}
```

---

## Пример использования (min-heap)

```cpp
#include <iostream>
#include <queue>
#include <vector>
#include <functional> // для std::greater

int main() {
    std::priority_queue<int, std::vector<int>, std::greater<int>> pq;
    pq.push(3);
    pq.push(5);
    pq.push(7);
    pq.push(2);

    std::cout << "Минимальный: " << pq.top() << '\n'; // 2
    pq.pop(); // удаляем 2
    std::cout << "Новый минимум: " << pq.top() << '\n'; // 3

    return 0;
}
```

## Очередь с приоритетом для пользовательских типов

Если вы храните не простые типы, а структуры, нужно либо определить оператор `<`, либо передать компаратор.

### Вариант 1: перегрузка оператора `<`

```cpp
struct Person {
    std::string name;
    int age;
};

// Чтобы priority_queue работала как max-heap по возрасту
bool operator<(const Person& a, const Person& b) {
    return a.age < b.age; // чем больше age, тем выше приоритет
}

// Использование:
std::priority_queue<Person> pq;
pq.push({"Alice", 30});
pq.push({"Bob", 25});
std::cout << pq.top().name; // Alice
```

### Вариант 2: передача компаратора

```cpp
struct CompareAge {
    bool operator()(const Person& a, const Person& b) const {
        return a.age < b.age; // для max-heap
    }
};

std::priority_queue<Person, std::vector<Person>, CompareAge> pq;
```

Для min-heap нужно использовать `>` в компараторе.

---

## Сложность операций и память

- Вставка `push` и удаление `pop` — **O(log n)** (поддержание свойства кучи).
- Получение вершины `top` — **O(1)**.
- Память — O(n) (дополнительных затрат почти нет, кроме самого контейнера).

---

## Когда использовать priority_queue?

- Алгоритмы, требующие постоянного извлечения наибольшего/наименьшего элемента: **алгоритм Дейкстры**, **алгоритм Прима**, **задачи с приоритетными заданиями**.
- Сортировка частичная (например, поиск k наибольших элементов).
- Симуляция событий с приоритетами.

Если же вам нужен не только доступ к экстремуму, но и поиск/удаление произвольных элементов, или обход всех элементов, лучше использовать `std::set`/`std::multiset`. Однако `priority_queue` имеет меньшие константы и использует непрерывную память (вектор), что часто даёт выигрыш в производительности.

---

## Отличия от std::set (упорядоченного множества)

| Характеристика          | `priority_queue`              | `std::set`                        |
|-------------------------|-------------------------------|-----------------------------------|
| Структура               | Куча (heap)                   | Сбалансированное дерево (красно-чёрное) |
| Доступ к макс/мин       | O(1) (только вершина)         | O(1) через `begin()`/`rbegin()`   |
| Вставка/удаление        | O(log n)                      | O(log n)                          |
| Поиск произвольного     | Нет (только вершина)          | O(log n)                          |
| Обход всех элементов    | Только извлечением (разрушает) | Да (итераторы)                    |
| Память                  | Непрерывная (вектор)          | Дерево (фрагментирована)          |
| Дубликаты               | Допускаются                   | `set` — нет, `multiset` — да      |

---

## 9. Важные замечания

- `priority_queue` **не поддерживает итераторы**, поэтому нельзя обойти его элементы без последовательного вызова `top()` и `pop()` (что разрушает очередь). Если нужен обход без удаления, используйте копирование очереди или другой контейнер.
- В `priority_queue` нет метода `clear()` — для очистки можно присвоить новую пустую очередь: `pq = std::priority_queue<int>();` или использовать `pq = {};` (C++11).
- При создании min-heap через `std::greater<T>` не забудьте включить `<functional>`.
