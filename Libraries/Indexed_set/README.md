# 📘 Политико-ориентированные структуры данных (GNU PBDS) — `indexed_set`

Компилятор GCC предоставляет расширение стандартной библиотеки — **Policy-Based Data Structures (PBDS)**. Они реализованы в пространстве имён `__gnu_pbds` и позволяют использовать более сложные структуры, такие как **деревья**, **хеш-таблицы** и **приоритетные очереди** с настраиваемыми политиками. Наиболее полезной для олимпиадного программирования является **`tree`**, которая ведёт себя как упорядоченное множество, но дополнительно поддерживает **индексацию** (порядковую статистику) — т.е. доступ к элементу по его позиции в отсортированном порядке и определение позиции элемента.

---

## Подключение и объявление

Для использования PBDS необходимо включить специальные заголовочные файлы и подключить пространство имён:

```cpp
#include <ext/pb_ds/assoc_container.hpp>
#include <ext/pb_ds/tree_policy.hpp>
using namespace __gnu_pbds;
```

Объявление `indexed_set` для типа `int`:

```cpp
typedef tree<int, null_type, less<int>, rb_tree_tag,
             tree_order_statistics_node_update> indexed_set;
```

Параметры шаблона:
- `int` — тип ключа.
- `null_type` — тип значения (для множества — `null_type`, для отображения — другой тип).
- `less<int>` — компаратор (по умолчанию по возрастанию).
- `rb_tree_tag` — тип дерева (красно-чёрное).
- `tree_order_statistics_node_update` — политика, добавляющая поддержку порядка (порядковой статистики).

Для мультимножества (с дубликатами) можно использовать `null_type` и тот же синтаксис, но нужно определить структуру, хранящую пару (ключ, уникальный идентификатор), либо использовать `tree` с парой (ключ, индекс), чтобы различать одинаковые ключи.

---

## Основные методы

`indexed_set` поддерживает все методы обычного `std::set`:
- `insert(key)` — вставка (O(log n)).
- `erase(key)` — удаление (O(log n)).
- `find(key)` — поиск (O(log n)).
- `size()`, `empty()`, `clear()` и т.д.

**Дополнительные методы (порядковая статистика):**

| Метод | Описание | Сложность |
|-------|----------|-----------|
| `find_by_order(k)` | Возвращает итератор на элемент, который находился бы на позиции `k` (0-индексация) в отсортированном порядке. Если `k >= size()`, возвращает `end()`. | O(log n) |
| `order_of_key(key)` | Возвращает количество элементов, строго меньших `key` (т.е. позицию, на которой оказался бы `key` при вставке). | O(log n) |

## Пример использования

```cpp
#include <iostream>
#include <ext/pb_ds/assoc_container.hpp>
#include <ext/pb_ds/tree_policy.hpp>

using namespace std;
using namespace __gnu_pbds;

typedef tree<int, null_type, less<int>, rb_tree_tag,
             tree_order_statistics_node_update> indexed_set;

int main() {
    indexed_set s;
    s.insert(2);
    s.insert(3);
    s.insert(7);
    s.insert(9);

    // find_by_order
    auto it = s.find_by_order(2); // элемент на позиции 2 (0-индексация)
    if (it != s.end())
        cout << "Элемент на позиции 2: " << *it << '\n'; // 7

    // order_of_key
    cout << "Позиция элемента 7: " << s.order_of_key(7) << '\n'; // 2
    cout << "Позиция элемента 6 (если бы был): " << s.order_of_key(6) << '\n'; // 2 (т.к. меньше 7, но больше 3)
    cout << "Позиция элемента 8: " << s.order_of_key(8) << '\n'; // 3

    // Проверка наличия
    if (s.find(5) == s.end())
        cout << "5 нет в множестве\n";

    // Удаление
    s.erase(3);

    return 0;
}
```

## Когда использовать indexed_set?

- **Задачи на порядковые статистики** — нужно часто находить k-й по порядку элемент или определять ранг элемента.
- **Динамические множества** — операции вставки/удаления и запросы порядка в реальном времени.
- **Альтернатива** использованию `std::set` + `std::advance` (которое работает за O(n)) или `std::vector` + сортировка (слишком медленно при частых изменениях).

---

## Особенности и ограничения

- Это расширение GCC, не входит в стандарт C++. Код не будет компилироваться в MSVC или Clang без дополнительных настроек.
- Для хранения дубликатов (мультимножества) можно использовать `tree<pair<int,int>, ...>` или определить свою структуру с уникальным идентификатором.
- Методы `find_by_order` и `order_of_key` работают за O(log n), что делает этот контейнер очень мощным.

## Пример задачи: поиск k-го по порядку

```cpp
// Дан массив, нужно выполнить запросы:
// 1. добавить число
// 2. удалить число
// 3. найти k-е по порядку
int main() {
    indexed_set s;
    int q;
    cin >> q;
    while (q--) {
        int type, x;
        cin >> type >> x;
        if (type == 1) s.insert(x);
        else if (type == 2) s.erase(x);
        else if (type == 3) {
            auto it = s.find_by_order(x - 1); // т.к. запрос может быть в 1-индексации
            if (it != s.end()) cout << *it << '\n';
            else cout << "Invalid\n";
        }
    }
}
```

## Сравнение с другими контейнерами

| Контейнер | Упорядоченность | Поиск по индексу | Вставка/удаление | Поиск по ключу | Стандарт |
|-----------|----------------|------------------|------------------|----------------|----------|
| `std::set` | Да | Нет (только обход) | O(log n) | O(log n) | C++98 |
| `std::vector` + сортировка | Да | O(1) (по индексу) | O(n) (вставка/удаление) | O(log n) (бинарный поиск) | C++98 |
| `indexed_set` (PBDS) | Да | O(log n) | O(log n) | O(log n) | GCC только |
