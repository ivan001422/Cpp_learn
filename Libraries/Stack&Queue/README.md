# 📘 Стек (stack) и очередь (queue)

Стек и очередь — это классические структуры данных, ограничивающие доступ к элементам по принципу **LIFO** (Last In, First Out) и **FIFO** (First In, First Out) соответственно. В C++ они реализованы как **адаптеры контейнеров** — то есть они не являются самостоятельными контейнерами, а предоставляют ограниченный интерфейс поверх базового контейнера (по умолчанию — `std::deque`). Это позволяет использовать их с различными базовыми контейнерами (`vector`, `deque`, `list`).

---

## Стек (`std::stack`)

Стек — структура, где добавление и удаление элементов происходит только с одного конца, называемого **вершиной** стека.

### Подключение и объявление
```cpp
#include <stack>
std::stack<int> st;          // пустой стек, базовый контейнер — deque<int>
std::stack<int, std::vector<int>> st2; // с использованием вектора
```

### Основные методы
| Метод | Описание | Сложность |
|-------|----------|-----------|
| `push(value)` | Добавляет элемент на вершину | O(1) |
| `pop()` | Удаляет верхний элемент (без возврата значения) | O(1) |
| `top()` | Возвращает ссылку на верхний элемент (не удаляя) | O(1) |
| `empty()` | Проверяет, пуст ли стек | O(1) |
| `size()` | Возвращает количество элементов | O(1) |

> **Важно:** В стеке нет итераторов, нельзя обойти элементы кроме как последовательным извлечением.

### Пример использования
```cpp
#include <iostream>
#include <stack>

int main() {
    std::stack<int> s;
    s.push(2);   // [2]
    s.push(5);   // [2, 5]
    s.push(7);   // [2, 5, 7]

    std::cout << "Верх: " << s.top() << '\n'; // 7
    s.pop();     // удаляем 7
    std::cout << "Новый верх: " << s.top() << '\n'; // 5

    while (!s.empty()) {
        std::cout << s.top() << ' ';
        s.pop();
    } // выведет "5 2"
    return 0;
}
```

### Когда использовать стек?
- Обработка вложенных структур (парсинг выражений, проверка скобок).
- Реализация отката (undo) в редакторах.
- Обход деревьев в глубину (DFS).
- Хранение временных данных в алгоритмах.

## Очередь (`std::queue`)

Очередь — структура, где элементы добавляются в конец, а извлекаются из начала (FIFO).

### Подключение и объявление
```cpp
#include <queue>
std::queue<int> q;          // база — deque<int>
std::queue<int, std::list<int>> q2; // с использованием списка
```

### Основные методы
| Метод | Описание | Сложность |
|-------|----------|-----------|
| `push(value)` | Добавляет элемент в конец очереди | O(1) |
| `pop()` | Удаляет первый элемент (без возврата) | O(1) |
| `front()` | Возвращает ссылку на первый элемент | O(1) |
| `back()` | Возвращает ссылку на последний элемент | O(1) |
| `empty()` | Проверяет, пуста ли очередь | O(1) |
| `size()` | Количество элементов | O(1) |

Как и стек, очередь не предоставляет итераторов.

### Пример использования
```cpp
#include <iostream>
#include <queue>

int main() {
    std::queue<int> q;
    q.push(2);   // [2]
    q.push(5);   // [2, 5]
    q.push(3);   // [2, 5, 3]

    std::cout << "Первый: " << q.front() << '\n'; // 2
    std::cout << "Последний: " << q.back() << '\n'; // 3
    q.pop();     // удаляем 2
    std::cout << "Новый первый: " << q.front() << '\n'; // 5

    while (!q.empty()) {
        std::cout << q.front() << ' ';
        q.pop();
    } // выведет "5 3"
    return 0;
}
```

### Когда использовать очередь?
- Обработка задач в порядке поступления (FIFO).
- Алгоритмы обхода графа в ширину (BFS).
- Буферизация данных, очереди сообщений.
- Планировщики задач.

## Адаптеры контейнеров – что это?

`std::stack` и `std::queue` не хранят данные самостоятельно, а делегируют хранение базовому контейнеру (`deque`, `vector`, `list`). Это даёт возможность выбрать подходящий контейнер в зависимости от требований к производительности.

По умолчанию используется `std::deque`, так как он обеспечивает эффективные операции вставки/удаления с обоих концов и произвольный доступ, хотя для стека и очереди достаточно операций только с одного конца.

### Выбор базового контейнера
- **`deque`** — универсален, хорош для стека и очереди.
- **`vector`** — может использоваться для стека (быстрый доступ к памяти), но при реаллокации может быть дороже.
- **`list`** — возможен для очереди, но обычно медленнее из-за фрагментации памяти.

Для явного указания базового контейнера используйте второй шаблонный параметр:
```cpp
std::stack<int, std::vector<int>> st;
std::queue<int, std::list<int>> q;
```

---

## Отличия от других контейнеров

| Характеристика      | `stack` / `queue`          | `vector` / `deque` / `list` |
|---------------------|----------------------------|-----------------------------|
| Интерфейс           | Ограниченный (только push/pop/top и т.д.) | Полный (итераторы, произвольный доступ) |
| Итераторы           | Отсутствуют                | Присутствуют                |
| Управление памятью  | Зависит от базового контейнера | Встроенное                |
| Основное назначение | Специализированные структуры (LIFO/FIFO) | Универсальные массивы/списки |

---

## Сложность операций (в среднем)

Все операции у стеков и очередей — **O(1)** при использовании базового контейнера с эффективными операциями на концах (`deque` или `list`). При использовании `vector` для стека `push` может быть O(1) амортизированно, а `pop` — O(1).

---

## Важные замечания

- `pop()` не возвращает значение, поэтому перед вызовом следует сохранить значение через `top()` или `front()`.
- При работе с пустыми стеками/очередями вызов `top()`/`front()` или `pop()` приводит к неопределённому поведению — всегда проверяйте `empty()`.
- Стек и очередь не поддерживают итераторы, поэтому для обхода всех элементов нужно последовательно извлекать их (что разрушает структуру). Если нужен обход без удаления, используйте базовый контейнер напрямую.
